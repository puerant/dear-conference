## 四、Bug 修复记录

### 4.1 BUG-SYS-001: Redis 缓存反序列化问题 ✅ 已修复

**问题描述**：
访问 `/api/admin/system/config` 接口时报错：
```
系统异常: uri=/api/admin/system/config, message=class java.util.LinkedHashMap cannot be cast to class com.huiwutong.conference.entity.OrderTaskConfig
```

**原因分析**：
- `GenericJackson2JsonRedisSerializer` 在反序列化时，会将复杂对象反序列化为 `LinkedHashMap`
- 尝试启用 `activateDefaultTyping` 后，旧缓存数据缺少 `@class` 属性导致新的序列化异常

**最终修复方案**：

1. 更换缓存 key 为 `system:order_task_config:v2`，绕过旧的不兼容数据
2. 保持原 RedisConfig 序列化配置不变
3. 服务层增加防御性类型检查：

```java
@Override
public OrderTaskConfig getConfig() {
    OrderTaskConfig config = null;

    try {
        Object cached = redisTemplate.opsForValue().get(CACHE_KEY);
        if (cached instanceof OrderTaskConfig) {
            config = (OrderTaskConfig) cached;
        } else if (cached != null) {
            // 缓存格式不兼容（如 LinkedHashMap），清除旧缓存
            redisTemplate.delete(CACHE_KEY);
        }
    } catch (Exception e) {
        // 序列化异常或其他异常，清除旧缓存
        redisTemplate.delete(CACHE_KEY);
    }

    if (config == null) {
        // 从数据库查询...
    }
    return config;
}
```

**修改文件**：
- `backend/src/main/java/com/huiwutong/conference/service/impl/OrderTaskConfigServiceImpl.java`

**验证结果**：后端编译成功，接口正常返回数据
