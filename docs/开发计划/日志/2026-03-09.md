## 四、问题与修复

### BUG-CONF-001：Admin API 路径重复 `/api` 前缀 ✅ 已修复

**现象**：
前端请求 `/api/admin/experts` 实际发送到 `/api/api/admin/experts`

**原因**：
- `request.ts` 配置 `baseURL: '/api'`
- API 函数中使用 `/api/admin/experts` 完整路径
- 导致路径重复

**修复方案**：
修改 `frontend/admin/src/api/conference.ts`，去掉 API 函数中的 `/api` 前缀：
- `/api/admin/conference` -> `/admin/conference`
- `/api/admin/experts` -> `/admin/experts`
- `/api/admin/schedule` -> `/admin/schedule`
- `/api/admin/hotels` -> `/admin/hotels`

**修复结果**：
API 请求路径正确，数据正常加载

### BUG-CONF-002：前端响应数据处理问题 ✅ 已修复

**现象**：
页面显示"系统繁忙"或数据不显示

**原因**：
`request.ts` 的响应拦截器已经解包了 `data`，但页面组件仍然使用 `res.data`

**修复方案**：
修改所有页面组件的数据处理方式：
```typescript
// 错误写法
const res = await getExpertList()
tableData.value = res.data || []

// 正确写法
const data = await getExpertList()
tableData.value = data || []
```

**修复结果**：
所有页面数据正常显示
