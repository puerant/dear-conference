### 架构验证（无问题 ✓）

| 检查点 | 结论 |
|--------|------|
| CORS 配置 | `WebConfig.java` 使用 `allowedOriginPatterns("*")`，开发环境无跨域问题 ✓ |
| API 地址 | Admin / Portal 均通过 `.env.development` 配置 `VITE_API_BASE_URL=http://localhost:8080/api` ✓ |
| Token 存储隔离 | Admin 用 `admin_token`，Portal 用 `portal_token`，两套 localStorage 互不干扰 ✓ |
| 响应格式 | 后端 `Result<T>` 与前端 `ApiResponse<T>` 字段完全一致（code/message/data/timestamp）✓ |
| 角色序列化 | `UserRole` 枚举 `@JsonValue` 标注在 `code` 字段，JSON 返回字符串（"speaker"/"admin"），前端 store 期望 `role: string` ✓ |
| 业务错误处理 | 后端 `GlobalExceptionHandler` 返回 HTTP 200 + `{code: 错误码}`，前端拦截器在 `success` 回调中识别非 200 code 并处理 ✓ |
| 注册角色限制 | `RegisterDto` 用 `@Pattern` 校验 `role` 只能是 `"speaker"` 或 `"attendee"`，前端注册表单选项与之一致 ✓ |
| 密码修改 | `UpdateUserDto` 包含 `oldPassword` + `newPassword` 字段，Portal 的 `changePassword` 调用 `PUT /user/profile` 路径正确 ✓ |

## 第五步：问题记录与修复（下午，约 1 小时）

### 常见联调问题排查清单

| 问题现象 | 可能原因 | 排查方式 |
|----------|----------|----------|
| 请求 CORS 错误 | 后端 `WebConfig` CORS 未配置前端域名 | 检查 `WebConfig.java` 允许的 origin |
| 401 / Token 无效 | 前端未携带 Token 或 Token 格式错误 | Network 面板查看请求头 |
| 接口 404 | 前端 baseURL 或接口路径拼写错误 | 对比 Controller 路径与 api/*.ts |
| 响应数据格式不匹配 | 后端返回字段名与前端 TypeScript 类型定义不一致 | 对比 `api/types.ts` 与后端 DTO |
| 分页数据为空 | PageResult 字段名不一致（total/list vs count/records） | 检查 `PageResult.java` 与前端解析 |
| 枚举显示异常 | `format.ts` 的枚举值映射与数据库 `item_value` 不一致 | 对比字典表数据与前端映射表 |
| 文件上传失败 | 文件大小超限或 Content-Type 未设置 | 检查 multipart 配置和前端请求头 |
