### 常见问题排查清单

| 问题现象 | 可能原因 | 排查方式 |
|----------|----------|----------|
| 验证码未收到/未在日志出现 | `dev-mode` 配置未生效 | 检查 `application-dev.yml` 中 `mail.dev-mode` 值 |
| 投稿状态标签显示「未知」 | format.ts key 仍为数字或枚举 code 拼写错误 | 对比 `format.ts` key 与后端 `SubmissionStatus.code` |
| 分配审稿人弹框空白 | `GET /api/admin/users?role=reviewer` 仍报错 | Network 面板查看该请求响应，确认 Bug-4 修复有效 |
| 审稿详情摘要为空 | `abstract` 字段映射问题 | 检查请求响应中是否存在 `abstract` 字段（非 `abstract0`） |
| 路由守卫无限跳转 | `beforeEach` 内 redirect 逻辑判断缺 `to.path !== '/login'` 条件 | 检查路由守卫 next 逻辑 |
| 401 后未触发 Token 刷新 | Axios 拦截器 `_retried` 标记未清除或刷新接口 URL 错误 | 检查 `request.ts` 响应拦截器实现 |

## 明日预告（Phase 6 Day 3）

明日任务：票务模块 + 订单模块 + 凭证模块联调

**核心流程**：
```
管理员创建票务（上架）
  → 参会者查看票务列表 → 购票下单
    → 模拟支付 → 订单状态变为「已支付」
      → 自动生成凭证（credentials 表）
        → 参会者查看凭证 + 二维码展示
          → 管理员核销凭证
            → 凭证状态变为「已使用」

  [异常流]
  → 订单超时未支付 → 定时任务自动取消（30分钟）
  → 参会者申请退款 → 管理员审核 → 凭证失效 + 库存恢复
```

**重点关注**：
- 票务库存并发扣减逻辑（`sold_count` 准确性）
- 凭证自动生成时机（支付回调触发 or 支付状态轮询）
- 二维码数据格式与核销接口的匹配
- 退款后凭证状态联动（`credentials.status` → 3 已过期）
- 定时任务取消订单的幂等性
