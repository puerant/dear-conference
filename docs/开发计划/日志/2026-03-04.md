## 二、Bug 修复记录

### BUG-EMAIL-001：前后端 boolean/Integer 类型不匹配 ✅ 已修复

**问题描述**：
保存邮箱配置时后端返回 500，日志报错：
```
JSON parse error: Cannot deserialize value of type `java.lang.Integer`
from Boolean value (token `JsonToken.VALUE_TRUE`)
```

**原因分析**：
- 后端 `EmailConfig.java` 的 `sslEnabled`、`isEnabled` 字段类型为 `Integer`
- 前端 `system.ts` 接口定义中对应字段类型为 `boolean`
- `el-switch` 组件默认返回 `true/false`，Jackson 无法将 Boolean 反序列化为 Integer

**相同问题还存在于**：
- `PaymentConfig`：`isEnabled`、`sandboxMode`
- `OrderTaskConfig`：`cancelTaskEnabled`、`refundTaskEnabled`

**修复方案**：在前端 API 发送层做类型转换，统一将 boolean 转为 0/1

**修改文件**：`frontend/admin/src/api/system.ts`

```typescript
// 邮箱配置
updateEmailConfig(data: EmailConfig) {
  return request.post('/admin/system/email-config', {
    ...data,
    sslEnabled: data.sslEnabled ? 1 : 0,
    isEnabled: data.isEnabled ? 1 : 0,
  })
},

// 支付配置
updatePaymentConfig(data: PaymentConfig) {
  return request.post('/admin/system/payment-config', {
    ...data,
    isEnabled: data.isEnabled ? 1 : 0,
    sandboxMode: data.sandboxMode ? 1 : 0,
  })
},

// 定时任务配置
updateOrderTaskConfig(data: OrderTaskConfig) {
  return request.post('/admin/system/order-task-config', {
    ...data,
    cancelTaskEnabled: data.cancelTaskEnabled ? 1 : 0,
    refundTaskEnabled: data.refundTaskEnabled ? 1 : 0,
  })
},
```

### BUG-EMAIL-002：AES 密钥长度非法导致加密失败 ✅ 已修复

**问题描述**：
前端类型修复后仍然 500，后端日志报错：
```
系统异常: uri=/api/admin/system/email-config, message=加密失败
java.lang.RuntimeException: 加密失败
```

**原因分析**：
- `AesUtil.java` 使用 `@Value("${app.config.secret:default-secret-key-16bytes}")` 读取密钥
- 默认值 `"default-secret-key-16bytes"` 实际为 **26 字节**
- AES 要求密钥长度必须为 **16、24 或 32 字节**，26 字节导致 `Cipher.init()` 抛出异常
- `application-dev.yml` 中未配置 `app.config.secret`，始终使用非法默认值

**修复方案**：在配置文件中设置合法的 32 字节 AES-256 密钥，并在 `application.yml` 中增加环境变量支持

**修改文件**：
- `backend/src/main/resources/application-dev.yml`
- `backend/src/main/resources/application.yml`

```yaml
# application-dev.yml 新增
app:
  config:
    secret: HuiWuTong2026@Conference-Secret!  # AES-256 密钥，必须 32 字节

# application.yml 新增
app:
  config:
    secret: ${APP_CONFIG_SECRET:HuiWuTong2026@Conference-Secret!}
```

**注意**：生产环境部署时应通过环境变量 `APP_CONFIG_SECRET` 设置独立的强密钥，不可使用默认值。

### BUG-EMAIL-003：getSystemConfig 响应缺少 username 字段 ✅ 已修复

**问题描述**：
配置保存成功后，页面重新加载时用户名字段为空，点保存触发「请输入用户名」校验失败。
`GET /api/admin/system/config` 返回的 `emailConfig` 对象中缺少 `username` 字段。

**原因分析**：
`SystemConfigServiceImpl.getAllConfigs()` 手动构建 `emailMap` 时漏写了 `username` 字段：

```java
// 修复前（漏写 username）
emailMap.put("id", emailConfig.getId());
emailMap.put("host", emailConfig.getHost());
emailMap.put("port", emailConfig.getPort());
// username 遗漏！
emailMap.put("fromName", emailConfig.getFromName());
```

**修复方案**：补充 `username` 字段

**修改文件**：`backend/src/main/java/com/huiwutong/conference/service/impl/SystemConfigServiceImpl.java`

```java
emailMap.put("id", emailConfig.getId());
emailMap.put("host", emailConfig.getHost());
emailMap.put("port", emailConfig.getPort());
emailMap.put("username", emailConfig.getUsername());  // 补充
emailMap.put("fromName", emailConfig.getFromName());
emailMap.put("fromEmail", emailConfig.getFromEmail());
emailMap.put("sslEnabled", emailConfig.getSslEnabled() != null && emailConfig.getSslEnabled() == 1);
emailMap.put("isEnabled", emailConfig.getIsEnabled() != null && emailConfig.getIsEnabled() == 1);
```

### BUG-EMAIL-004：编辑时密码必填校验阻止「留空保留原密码」操作 ✅ 已修复

**问题描述**：
配置页面回填后，密码框为空（安全设计，不回显密码）。
直接点「保存配置」触发必填校验「请输入密码」，无法在不修改密码的情况下更新其他配置项。

**原因分析**：
- 后端 `saveOrUpdate()` 已支持：密码为空时保留数据库中已有的加密密码
- 但前端 `email.vue` 的校验规则 `password: [{ required: true }]` 对新增和编辑场景一视同仁

**修复方案**：
1. 将 `rules` 改为 `computed`，根据 `form.id` 是否存在动态控制密码是否必填
2. 密码框 placeholder 根据场景动态显示

**修改文件**：`frontend/admin/src/views/system/email.vue`

```typescript
// rules 改为 computed
const rules = computed<FormRules>(() => ({
  // ...
  password: [{ required: !form.id, message: '请输入密码', trigger: 'blur' }],
  // ...
}))
```

```html
<!-- placeholder 动态提示 -->
<el-input v-model="form.password" type="password" show-password
  :placeholder="form.id ? '留空则不修改密码' : '请输入密码'" />
```

**同时在 loadConfig 中补充 boolean 转换**（BUG-EMAIL-001 关联修复）：

```typescript
// email.vue loadConfig
form.sslEnabled = Boolean(res.emailConfig.sslEnabled)
form.isEnabled = Boolean(res.emailConfig.isEnabled)
```
