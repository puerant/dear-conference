## 二、核心业务流程

### 2.1 个人票凭证生成

```
订单支付成功
  → OrderServiceImpl.confirmPaid() / PaymentServiceImpl.confirmOrderPaid()
      orderType != 2（非团体票）时：
      → credentialService.generateCredential(orderId)
          ① 幂等查询：SELECT WHERE order_id=? AND group_member_id IS NULL
          ② 若已存在则直接返回（防重复生成）
          ③ buildCredentialNo()：CRED{timestamp}{random}
          ④ assignSeat(ticketId)：COUNT 同票种凭证 +1 → SN-{seq:04d}
          ⑤ CredentialQrUtil.toBase64Png(credentialNo)：生成 300×300 二维码 Base64 PNG
          ⑥ expire_at = NOW() + credential.expire-days（默认 365 天）
          ⑦ INSERT INTO credentials (order_id, credential_no, qr_code, seat_info, status=VALID, expire_at)
             group_member_id = NULL
```

### 2.2 团体成员凭证生成

```
成员填写信息后（GroupOrderServiceImpl.addMember / updateMember）
  → credentialService.generateCredentialForMember(orderId, groupMemberId)
      ① 幂等查询：SELECT WHERE group_member_id=?
      ② 若已存在则直接返回
      ③ buildCredentialNo()：CRED{timestamp}{random}
      ④ assignSeat(ticketId)：同票种顺序座位
      ⑤ CredentialQrUtil.toBase64Png(credentialNo)：二维码 Base64 PNG
      ⑥ expire_at = NOW() + credential.expire-days
      ⑦ INSERT INTO credentials (order_id, group_member_id, credential_no, qr_code, seat_info, status=VALID, expire_at)

⚠️ 团体票整单不生成凭证，凭证以成员（group_member_id）为粒度
```

### 2.3 个人票凭证查询（含归属校验）

```
Portal 参会者 → 点击订单列表「凭证」按钮（仅已支付订单显示）
  → GET /api/credentials/order/{orderId}（携带 attendee token）
  → CredentialServiceImpl.getByOrderId(orderId, userId)
      ① orderMapper.selectById(orderId) 查询订单
      ② order == null → ORDER_NOT_FOUND(3001)
      ③ order.userId != userId → CREDENTIAL_ACCESS_DENIED(6004)
      ④ 查询 credentials WHERE order_id=? AND group_member_id IS NULL
      ⑤ 不存在 → CREDENTIAL_NOT_FOUND(6001)
  → 返回 Credential（credentialNo、qrCode[Base64]、seatInfo、status、expireAt）

前端展示二维码：
  <img :src="'data:image/png;base64,' + credential.qrCode" />
```

### 2.4 凭证核销（含有效期校验）

```
Admin 前端 → 输入凭证号 → 点击「核销」
  → POST /api/credentials/{credentialNo}/use
  → CredentialServiceImpl.useCredential()
      ① 查询凭证（CREDENTIAL_NOT_FOUND 6001）
      ② status = USED → CREDENTIAL_ALREADY_USED(6002)（幂等保护）
      ③ status = EXPIRED → CREDENTIAL_EXPIRED(6003)
      ④ expire_at != null && now > expire_at
           → 同步置为 EXPIRED + 抛 CREDENTIAL_TIME_EXPIRED(6005)
      ⑤ UPDATE credentials SET status=USED, used_at=NOW()
  → 返回 "核销成功"
```

### 2.5 凭证有效期自动过期

```
CredentialExpireTask（@Scheduled cron = "0 0 1 * * ?"，每天凌晨 1:00）
  → credentialMapper.expireOutdated(now)
      UPDATE credentials SET status=3（EXPIRED）
        WHERE status=1（VALID）
          AND expire_at IS NOT NULL
          AND expire_at < NOW()
  → 日志记录：本次共过期凭证 N 张

配置：application.yml
  credential.expire-days: 365（支持 ${CREDENTIAL_EXPIRE_DAYS} 环境变量覆盖）
```

### 2.6 负责人重新生成成员凭证

```
Portal 团体管理页 → 点击成员行「凭证」按钮
  → POST /api/orders/{id}/group/members/{memberId}/credential
  → GroupOrderServiceImpl.regenerateCredential()
      ① 校验订单归属
      ② 校验成员归属于该订单
      ③ 校验成员 status = 2（已填写）
      ④ credentialService.expireByGroupMemberId(memberId)  → 旧凭证 EXPIRED
      ⑤ credentialService.generateCredentialForMember(orderId, memberId)
         → 新凭证（含新二维码、新座位号、新有效期）
```

### 2.7 退款触发凭证失效

```
退款时：credentialService.expireByOrderId(orderId)
  → SELECT WHERE order_id=? （仅匹配 group_member_id IS NOT NULL 的也会被查到）
  → 若 status=VALID → UPDATE SET status=EXPIRED
团体票退款：DELETE FROM group_members（成员槽清空，群体凭证随之失去业务意义）
```

### 2.8 座位分配逻辑

```
assignSeat(ticketId):
  ① SELECT COUNT(*) FROM credentials c JOIN orders o ON c.order_id=o.id
       WHERE o.ticket_id = #{ticketId}
  ② seat_info = String.format("SN-%04d", count + 1)

示例：同一票种第 1 张凭证 → SN-0001，第 10 张 → SN-0010

注意：COUNT 在事务内执行，低并发场景下顺序准确；
      高并发场景下（多线程同时支付）可能产生重复座位号（已知局限，见第六节）
```

### 2.9 状态流转图

```
VALID(1) ──[useCredential()]────────────────────────────> USED(2)
VALID(1) ──[expireByOrderId()，退款触发]────────────────> EXPIRED(3)
VALID(1) ──[expireByGroupMemberId()，清除成员]──────────> EXPIRED(3)
VALID(1) ──[expireByGroupMemberId()，重新生成前]─────────> EXPIRED(3) → 新 VALID 凭证
VALID(1) ──[expire_at 超期 + useCredential()触发]────────> EXPIRED(3) 并抛 6005
VALID(1) ──[CredentialExpireTask 定时扫描]───────────────> EXPIRED(3)

USED(2) / EXPIRED(3) 均为终态，不可逆转
```

## 三、重要实现细节

### 3.1 凭证生成时机

| 订单类型 | 凭证生成时机 | 生成方式 | group_member_id |
|---------|------------|---------|----------------|
| 个人票 | 支付成功时 | `generateCredential(orderId)` | NULL |
| 团体票（每位成员） | 成员填写/修改信息时 | `generateCredentialForMember(orderId, memberId)` | 成员 ID |

### 3.2 二维码字段规范

| 字段 | 值示例 | 说明 |
|------|--------|------|
| `credentialNo` | `CRED1772264638978977` | 凭证唯一标识，核销时使用 |
| `qrCode` | `iVBORw0KGgo...（Base64）` | 300×300 PNG 二维码图片 Base64 编码，内容为 `credentialNo` |

前端渲染方式：
```html
<img :src="`data:image/png;base64,${credential.qrCode}`" alt="入场凭证二维码" />
```

### 3.3 座位号格式

格式：`SN-{4位序列号}`

- 同一票种（ticketId 相同）按全局凭证总数顺序递增
- 个人票和团体成员票共享同一票种的座位计数器
- 示例：某团体票共 10 人，座位从 `SN-0001` 到 `SN-0010`

### 3.4 有效期配置

```yaml
# application.yml
credential:
  expire-days: 365  # 默认 365 天

# 可通过环境变量覆盖（生产环境）
CREDENTIAL_EXPIRE_DAYS=180
```

有效期设置时机：凭证生成时，`expire_at = LocalDateTime.now().plusDays(expireDays)`

### 3.5 幂等性设计

| 场景 | 幂等条件 | 行为 |
|------|---------|------|
| `generateCredential(orderId)` | `WHERE order_id=? AND group_member_id IS NULL` 已存在 | 直接返回已有凭证 |
| `generateCredentialForMember(orderId, memberId)` | `WHERE group_member_id=?` 已存在 | 直接返回已有凭证 |
| `useCredential(credentialNo)` | status=USED | 抛 CREDENTIAL_ALREADY_USED(6002) |
| `expireByOrderId(orderId)` | status != VALID | 不执行 UPDATE |
| `expireByGroupMemberId(memberId)` | status != VALID | 不执行 UPDATE |

## 五、数据库变更

### 5.1 credentials 表新增字段（Migration SQL）

```sql
-- 新增凭证有效期字段
ALTER TABLE credentials
    ADD COLUMN expire_at DATETIME NULL COMMENT '凭证有效期（null 表示永不过期）' AFTER used_at;
```

> **注意**：已存在的凭证 `expire_at` 为 NULL，定时任务不会过期这些记录（WHERE expire_at IS NOT NULL）。运营侧若需对历史凭证设置有效期，需手动 UPDATE。

## 六、已知局限

| 功能 | 说明 |
|------|------|
| 座位分配并发安全 | `assignSeat()` 使用 COUNT 后 +1，高并发场景下（多线程同时支付）可能分配重复座位号。如需严格唯一，应使用数据库行锁或独立 seat_counter 表 |
| 座位号与实体座位无关 | `SN-XXXX` 仅为顺序流水号，不对应实际场馆座位排布 |
| 批量核销 | 无批量核销接口，每次仅核销单张 |
| 有效期精度 | `expire_at` 精确到秒；定时任务每天凌晨 1:00 执行，超期后最长 ~23 小时才被定时任务清理（`useCredential()` 会实时检测并同步置为 EXPIRED）|
| 团体票 expireByOrderId 覆盖 | 退款时 `expireByOrderId` 仅过期 `group_member_id IS NULL` 的个人票凭证；团体成员凭证通过 DELETE group_members 间接清理 |

## 七、错误码

| 错误码 | 错误信息 | 触发场景 |
|--------|---------|---------|
| 3001 | 订单不存在 | `getByOrderId` 查询不到订单 |
| 6001 | 凭证不存在 | 按 orderId 查询无凭证、核销不存在的凭证号 |
| 6002 | 凭证已使用 | 对 status=USED 重复核销 |
| 6003 | 凭证已过期 | 对 status=EXPIRED 执行核销（含退款后凭证） |
| 6004 | 无权访问该凭证 | `getByOrderId` 时 orderId 不属于当前 userId |
| 6005 | 凭证已超过有效期 | 核销时实时检测 expire_at 已过 |
| 3005 | 无权访问该订单 | 重新生成凭证时订单不属于当前用户 |
| 422 | 参数错误 | 对未填写成员发起凭证重新生成 |
