## 三、核心业务流程

### 3.1 购票下单

```
前端 Portal /tickets → 点击「立即购票」填写参会人信息
  → POST /api/orders { ticketId, quantity, attendeeName, attendeeEmail }
  → @RequireRole("attendee") 权限校验
  → OrderServiceImpl.create()
      ① 查票务是否存在（TICKET_NOT_FOUND 4001）
      ② 检查票务状态 status=1（TICKET_DISABLED 4002）
      ③ 检查库存 stock >= quantity（TICKET_NOT_AVAILABLE 3003）
      ④ 原子扣减库存：
         UPDATE tickets SET stock = stock - #{quantity},
                            sold_count = sold_count + #{quantity}
         WHERE id = #{id} AND stock >= #{quantity}
         → 返回 0 行时抛 TICKET_NOT_AVAILABLE（防超卖）
      ⑤ 生成订单号：ORD{timestamp}{random}
      ⑥ INSERT INTO orders (status=UNPAID)
  → 返回 Order 对象

⚠️ 关键设计：库存在下单时即扣减（sold_count+1），取消时回滚（sold_count-1）
   不同于文档中的「支付后扣库存」描述 — 实际代码在 create() 中已扣减
```

### 3.2 订单支付（模拟）

```
前端 Portal /attendee/orders → 点击「支付」
  → POST /api/orders/{id}/pay
  → OrderServiceImpl.pay()
      ① 校验订单归属（ORDER_ACCESS_DENIED 3005）
      ② 校验状态 = UNPAID（ORDER_STATUS_ERROR 3002）
      ③ UPDATE orders SET status=PAID, paid_at=NOW()
      ④ 调用 credentialService.generateCredential(orderId)
         → 幂等生成：若已存在则直接返回
         → 凭证号：CRED{timestamp}{random}
         → QR 码内容：QR:{credentialNo}
         → INSERT INTO credentials (status=VALID)
  → 返回 "支付成功"

⚠️ 当前为模拟支付，无第三方支付网关，点击即成功
⚠️ 支付成功不改变 tickets.sold_count（已在下单时扣减）
```

### 3.3 订单取消

```
前端 Portal → 点击「取消」
  → POST /api/orders/{id}/cancel
  → OrderServiceImpl.cancel()
      ① 校验订单归属（ORDER_ACCESS_DENIED 3005）
      ② 校验状态 = UNPAID（ORDER_STATUS_ERROR 3002，已支付/已取消/已退款均不可取消）
      ③ UPDATE orders SET status=CANCELLED
      ④ 恢复库存：ticketMapper.increaseStock(ticketId, quantity)
         → stock+quantity, sold_count-quantity
      ⑤ 若为团体订单（orderType=2）：
         DELETE FROM group_members WHERE order_id = ?
         （UNPAID 阶段成员不会有凭证，无需处理凭证）
  → 返回成功
```

### 3.4 凭证查询

```
前端 Portal /attendee/credentials（或订单列表「凭证」按钮）
  → GET /api/credentials/order/{orderId}
  → CredentialServiceImpl.getByOrderId()
      - 查询 credentials WHERE order_id = ?
      - 不存在抛 CREDENTIAL_NOT_FOUND(6001)
  → 返回 Credential（含 credentialNo、qrCode、status 等）
```

### 3.5 凭证核销

```
Admin 前端 /credential → 输入凭证号 → 点击「核销」
  → POST /api/credentials/{credentialNo}/use
  → @RequireRole("admin") 权限校验
  → CredentialServiceImpl.useCredential()
      - 查询凭证（CREDENTIAL_NOT_FOUND 6001）
      - 已使用：抛 CREDENTIAL_ALREADY_USED(6002)（幂等保护）
      - 已过期：抛 CREDENTIAL_EXPIRED(6003)
      - UPDATE credentials SET status=USED, used_at=NOW()
  → 返回 "核销成功"
```

### 3.6 管理员退款

```
Admin 前端 /order/detail → 点击「退款」
  → POST /api/admin/orders/{id}/refund
  → @RequireRole("admin") 权限校验
  → OrderServiceImpl.refund()
      ① 校验状态 = PAID（ORDER_STATUS_ERROR 3002）
      ② UPDATE orders SET status=REFUNDED
      ③ credentialService.expireByOrderId(orderId)
         → 若凭证存在且状态为 VALID：UPDATE credentials SET status=EXPIRED
         → 团体票成员凭证以 order_id 关联，同样被过期处理
      ④ 恢复库存：ticketMapper.increaseStock(ticketId, quantity)
      ⑤ 若为团体订单（orderType=2）：
         DELETE FROM group_members WHERE order_id = ?
  → 返回 "退款成功"
```

### 3.7 Admin 凭证列表

```
Admin 前端 /credential → 查看凭证列表
  → GET /api/admin/credentials?page=1&pageSize=10&credentialNo=&status=
  → @RequireRole("admin") 权限校验
  → CredentialServiceImpl.listAll()
      - 支持按凭证号模糊查询、按状态精确筛选
      - fillOrderInfo()：批量关联订单/票务信息（orderNo、attendeeName、attendeeEmail、ticketName）
  → 返回 PageResult<Credential>
```

### 3.8 状态流转图

```
订单状态：
UNPAID(1) ──[pay()]──> PAID(2) ──[refund()]──> REFUNDED(4)
UNPAID(1) ──[cancel()]──> CANCELLED(3)

凭证状态：
VALID(1) ──[useCredential()]──> USED(2)
VALID(1) ──[expireByOrderId()，退款触发]──> EXPIRED(3)
```

## 四、重要实现细节

### 4.1 库存扣减时机

与常见「支付扣库存」设计不同，本系统**在下单时即扣减库存**：

| 操作 | tickets.sold_count 变化 | 时机 |
|------|------------------------|------|
| 下单 | `+quantity` | `OrderServiceImpl.create()` |
| 取消 | `-quantity` | `OrderServiceImpl.cancel()` |
| 支付 | 不变（已在下单时扣） | — |
| 退款 | `-quantity` | `OrderServiceImpl.refund()` |

> **测试时注意**：TC-ORD-001 验证下单后 `tickets.sold_count` 应 **变化**（不是「不变」）

### 4.2 管理员订单列表 status 过滤机制

`AdminOrderController.list()` → `OrderService.listAll()` → `applyStatusFilter()`

```java
// 接收字符串参数，用 Integer.parseInt() 解析为枚举值
OrderStatus orderStatus = OrderStatus.ofValue(Integer.parseInt(status));
```

前端 Admin 发送 `status=unpaid`（字符串 code），后端 `parseInt("unpaid")` 会抛 `NumberFormatException`！

> **已修复（Bug-21）**：联调时确认 Admin 订单列表状态筛选传字符串 code 会 500，已在 `applyStatusFilter()` 中增加 try-catch 回退逻辑。

### 4.3 Portal 订单状态类型不匹配

| 位置 | 问题 |
|------|------|
| `frontend/portal/src/api/types.ts` | `Order.status: number`，但后端 `@JsonValue` 返回字符串 `"unpaid"` |
| `frontend/portal/src/utils/format.ts` | `getOrderStatusLabel(status: number)` 使用数字 key `{1: '待支付'}` |
| `frontend/portal/src/views/order/List.vue` | `v-if="row.status === 1"` 与字符串比较，支付/取消按钮不显示 |

> **已修复（Bug-19）**：联调时确认状态全显示「未知」、操作按钮不可见，已将 `types.ts`/`format.ts`/`List.vue` 全部改用字符串类型。

### 4.4 TicketVo.available 双重计算

`decreaseStock()` 同时递减 `stock` 并递增 `sold_count`，导致 `available = stock - sold_count` 少计一份。

> **已修复（Bug-22）**：`TicketServiceImpl.convertToVo()` 中 `available` 改为直接取 `stock`。

## 七、已知局限 / 未实现功能

| 功能 | 说明 |
|------|------|
| 真实支付网关 | 当前为模拟支付，点击即成功；无 Alipay/WeChat Pay 对接 |
| 订单超时自动取消 | 无 `@Scheduled` 定时任务；超时取消场景无法自动触发 |
| 凭证-订单 userId 鉴权 | `GET /api/credentials/order/{orderId}` 仅校验 attendee 角色，不校验 orderId 是否属于当前用户 |
| 多票凭证 | 每个订单生成一张凭证（不按 quantity 生成多张） |
| 座位分配 | `credentials.seat_info` 字段存在，但无座位分配逻辑，始终为 null |
| 库存时机说明 | 下单即扣库存（非支付后扣），取消/退款均回滚，与日计划文档描述不一致 |

## 九、关联错误码

| 错误码 | 错误信息 | 触发场景 |
|--------|---------|---------|
| 3001 | 订单不存在 | 查询/支付/取消不存在的订单 ID |
| 3002 | 订单状态错误 | 支付已支付订单、取消非 UNPAID 订单、退款非 PAID 订单 |
| 3003 | 票务库存不足 | 下单时库存不足或原子操作返回 0 行 |
| 3005 | 无权访问该订单 | 查询/支付/取消他人订单 |
| 4001 | 票务不存在 | 下单时 ticketId 不存在 |
| 4002 | 票务已下架 | 下单时票务 status=0 |
| 6001 | 凭证不存在 | 按 orderId 查询无凭证（未支付）、核销不存在的凭证号 |
| 6002 | 凭证已使用 | 对 status=USED 的凭证重复核销 |
| 6003 | 凭证已过期 | 对 status=EXPIRED 的凭证执行核销（含退款后凭证） |
