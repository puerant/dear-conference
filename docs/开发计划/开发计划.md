## 三、详细计划

### Phase 1：后端基础层（3 天）

**目标**：搭建可运行的 Spring Boot 项目骨架，完成所有通用组件。

#### Day 1：项目配置 + 实体类

| 任务 | 文件 | 说明 |
|------|------|------|
| 配置 application.yml | `src/main/resources/application.yml` | MySQL、Redis、JWT 配置 |
| 配置 application-dev.yml | `src/main/resources/application-dev.yml` | 开发环境配置 |
| 初始化数据库 | `docs/sql/init.sql` | 执行建表 + 字典初始化 SQL |
| 创建 Entity 类（8张表） | `entity/` | User、Submission、Review、Ticket、Order、Credential、Dictionary、DictionaryItem |
| 创建 Mapper 接口 | `mapper/` | 继承 BaseMapper，8个接口 |

#### Day 2：通用工具类

| 任务 | 文件 | 说明 |
|------|------|------|
| Result 统一响应 | `common/vo/Result.java` | code、message、data、timestamp |
| PageResult 分页响应 | `common/vo/PageResult.java` | list、total、page、pageSize |
| BusinessException | `common/exception/BusinessException.java` | 业务异常类 |
| ErrorCode 枚举 | `common/exception/ErrorCode.java` | 各模块错误码 1001-6007 |
| GlobalExceptionHandler | `common/exception/GlobalExceptionHandler.java` | 全局异常捕获 |
| JwtUtil | `common/util/JwtUtil.java` | Token 生成/解析/验证 |
| PasswordUtil | `common/util/PasswordUtil.java` | BCrypt 加密封装 |
| OrderNoUtil | `common/util/OrderNoUtil.java` | 订单号/凭证号生成 |

#### Day 3：安全过滤器 + 注解

| 任务 | 文件 | 说明 |
|------|------|------|
| JwtAuthenticationFilter | `common/filter/JwtAuthenticationFilter.java` | 解析 Token，写入请求属性 |
| RequireLogin 注解 | `common/annotation/RequireLogin.java` | 标记需要登录的接口 |
| RequireRole 注解 | `common/annotation/RequireRole.java` | 标记需要特定角色的接口 |
| AuthInterceptor | `common/interceptor/AuthInterceptor.java` | 拦截器实现权限校验 |
| WebMvcConfig | `config/WebMvcConfig.java` | 注册拦截器、CORS 配置 |
| MybatisPlusConfig | `config/MybatisPlusConfig.java` | 分页插件配置 |

## 六、技术风险与应对

| 风险点 | 风险等级 | 应对方案 |
|--------|----------|----------|
| 支付接口集成 | 高 | 先实现模拟支付，预留真实接口扩展点 |
| 二维码生成库选型 | 低 | 使用成熟的 zxing 库 |
| 文件上传大文件 | 中 | 分片上传 + 10MB 前端限制 |
| 超时订单自动取消 | 中 | Spring Scheduler 定时任务，注意幂等性 |
| Redis 缓存雪崩 | 低 | 字典数据设置随机过期时间偏移 |
| 邮件发送延迟 | 低 | 异步处理（@Async），失败重试 |
| 邮箱配置安全性 | 中 | 密码加密存储（AES），不在日志中输出 |
| 日志数据量增长 | 中 | 定期归档历史日志，保留最近 90 天 |
| 定时任务并发 | 低 | 使用分布式锁（Redis）防止重复执行 |

## 七、开发规范提醒

- **后端**：阿里巴巴 Java 开发手册规范，Controller 不写业务逻辑
- **前端**：TypeScript 严格模式，所有 API 调用必须有类型定义
- **数据库**：无外键约束，应用层维护数据完整性
- **安全**：密码 BCrypt 加密，配置信息 AES 加密，JWT 24小时过期，SQL 参数化
- **日志**：敏感信息（密码、密钥）不得记录到日志，错误日志堆栈信息脱敏
- **定时任务**：使用分布式锁（Redis）防止并发执行，任务执行必写日志
- **Git**：遵循 `feat/fix/docs/style/refactor/test/chore` 提交规范
- **API**：统一使用 `Result<T>` 和 `PageResult<T>` 响应格式
