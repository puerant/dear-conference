# 学术会议网站需求文档

## 1. 项目概述

### 1.1 项目背景
本项目旨在构建一个综合性的学术会议入口及管理平台，为会议组织方、投稿讲者、审稿人、参会者提供一站式服务。

### 1.2 项目目标
- 提供便捷的会议投稿与管理功能
- 实现参会注册与票务管理
- 提供高效的审稿流程
- 提升会议组织管理效率

## 2. 用户角色及职责

### 2.1 投稿讲者
- 在网站注册账号并登录
- 创建和管理自己的投稿
- 查看投稿状态
- 收到投稿审核结果邮件通知

### 2.2 审稿人
- 在网站注册账号并登录
- 查看待审稿件列表
- 对稿件进行评审并提交评审意见

### 2.3 参会者
- 在网站注册账号并登录
- 选择并购买会议门票
- 查看订单状态
- 收到订单确认邮件
- 获取会议入会凭证
- 使用凭证参加会议

### 2.4 管理员
- 查看和管理所有订单信息
- 查看和管理所有投稿信息
- 管理用户账号
- 配置会议信息
- 管理票务类型

## 3. 功能需求

### 3.1 用户认证模块

#### 3.1.1 用户注册
- 支持邮箱注册
- 邮箱验证功能
- 设置密码及确认密码
- 选择用户角色（讲者/审稿人/参会者）

#### 3.1.2 用户登录
- 支持邮箱/密码登录
- 记住密码功能
- 找回密码功能（邮箱验证）

#### 3.1.3 用户信息管理
- 修改个人基本信息
- 修改密码
- 退出登录

### 3.2 投稿管理模块（讲者）

#### 3.2.1 创建投稿
- 填写投稿标题
- 填写投稿摘要
- 上传论文附件（支持PDF格式）
- 选择投稿分类/议题
- 添加作者信息
- 提交投稿

#### 3.2.2 投稿列表
- 查看所有投稿列表
- 显示投稿状态（待审核/审核中/已录用/已拒稿）
- 按状态筛选投稿

#### 3.2.3 投稿详情
- 查看投稿详细信息
- 查看审稿意见
- 查看审核结果

#### 3.2.4 删除投稿
- 删除待审核或审核中的投稿
- 删除确认提示

#### 3.2.5 邮件通知
- 投稿提交成功通知
- 审稿状态变更通知
- 录用/拒稿结果通知

### 3.3 审稿模块（审稿人）

#### 3.3.1 待审稿件列表
- 查看待审核稿件
- 显示稿件基本信息
- 按优先级排序

#### 3.3.2 审稿详情
- 查看投稿完整内容
- 下载论文附件
- 查看作者信息

#### 3.3.3 提交审稿意见
- 填写审评意见
- 选择录用/拒稿/修改
- 添加具体修改建议
- 提交审评结果

#### 3.3.4 审稿历史
- 查看历史审稿记录
- 查看已提交的审评意见

### 3.4 参会注册模块（参会者）

#### 3.4.1 票务选择
- 查看所有可用票务类型
- 显示票价及权益说明
- 选择票务数量

#### 3.4.2 创建订单
- 填写参会人信息
- 填写发票信息（可选）
- 确认订单信息
- 提交订单

#### 3.4.3 订单支付
- 支持多种支付方式（微信/支付宝/银行卡）
- 支付状态实时更新
- 支付成功跳转

#### 3.4.4 订单列表
- 查看所有订单
- 显示订单状态（待支付/已支付/已取消）
- 按状态筛选订单

#### 3.4.5 订单详情
- 查看订单详细信息
- 查看票务信息
- 查看参会人信息
- 查看发票信息

#### 3.4.6 入会凭证
- 查看入会凭证（二维码/电子票）
- 下载凭证（图片/PDF）
- 凭证包含：姓名、票务类型、座位信息（如有）

#### 3.4.7 邮件通知
- 订单创建成功通知
- 支付成功通知
- 入会凭证发放通知
- 会议提醒通知

### 3.5 管理后台模块（管理员）

#### 3.5.1 仪表盘
- 显示核心数据统计
  - 总投稿数
  - 审核中投稿数
  - 总订单数
  - 总销售额
  - 注册用户数

#### 3.5.2 投稿管理
- 查看所有投稿列表
- 按状态/分类/时间筛选
- 查看投稿详情
- 分配审稿人
- 手动审核投稿
- 导出投稿数据

#### 3.5.3 订单管理
- 查看所有订单列表
- 按状态/时间/票务类型筛选
- 查看订单详情
- 修改订单状态
- 退款处理
- 导出订单数据

#### 3.5.4 用户管理
- 查看所有用户列表
- 按角色/注册时间筛选
- 查看用户详情
- 修改用户角色
- 禁用/启用用户账号

#### 3.5.5 票务管理
- 创建票务类型
- 编辑票务信息（名称、价格、描述、库存）
- 启用/禁用票务
- 查看票务销售情况

#### 3.5.6 会议信息管理
- 编辑会议基本信息
  - 会议名称
  - 会议时间
  - 会议地点
  - 会议简介
- 上传会议Logo和海报

#### 3.5.7 审稿人管理
- 添加审稿人
- 分配审稿人账号
- 设置审稿人权限
- 查看审稿人工作量统计

## 4. 非功能性需求

### 4.1 性能需求
- 页面加载时间不超过2秒
- 支持并发用户数：≥1000
- 数据库查询响应时间不超过500ms

### 4.2 安全需求
- 用户密码加密存储（使用bcrypt或类似算法）
- 实施HTTPS加密传输
- 防SQL注入、XSS等常见安全攻击
- 实施RBAC权限控制
- 敏感操作需二次确认

### 4.3 可用性需求
- 系统可用性达到99.5%
- 提供7×24小时服务
- 数据定期备份（每日）

### 4.4 兼容性需求
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 支持移动端访问（响应式设计）
- 支持iOS/Android设备

## 5. 数据需求

### 5.1 核心数据实体

#### 用户表 (users)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键ID |
| email | varchar | 邮箱（唯一） |
| password | varchar | 加密密码 |
| name | varchar | 姓名 |
| role | enum | 角色(speaker/reviewer/attendee/admin) |
| status | enum | 状态(active/disabled) |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

#### 投稿表 (submissions)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键ID |
| user_id | bigint | 讲者ID |
| title | varchar | 投稿标题 |
| abstract | text | 摘要 |
| category | varchar | 投稿分类 |
| file_url | varchar | 附件URL |
| status | enum | 状态(pending/reviewing/accepted/rejected) |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

#### 审稿记录表 (reviews)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键ID |
| submission_id | bigint | 投稿ID |
| reviewer_id | bigint | 审稿人ID |
| comment | text | 审稿意见 |
| result | enum | 审稿结果(accept/reject/revise) |
| created_at | datetime | 创建时间 |

#### 票务表 (tickets)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键ID |
| name | varchar | 票务名称 |
| price | decimal | 价格 |
| description | text | 描述 |
| stock | int | 库存 |
| sold_count | int | 已售数量 |
| status | enum | 状态(active/disabled) |

#### 订单表 (orders)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键ID |
| order_no | varchar | 订单号 |
| user_id | bigint | 用户ID |
| ticket_id | bigint | 票务ID |
| quantity | int | 数量 |
| total_amount | decimal | 总金额 |
| status | enum | 状态(unpaid/paid/cancelled/refunded) |
| attendee_name | varchar | 参会人姓名 |
| attendee_email | varchar | 参会人邮箱 |
| created_at | datetime | 创建时间 |
| paid_at | datetime | 支付时间 |

#### 入会凭证表 (tickets_credentials)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键ID |
| order_id | bigint | 订单ID |
| credential_no | varchar | 凭证号 |
| qr_code | varchar | 二维码数据 |
| seat_info | varchar | 座位信息 |
| status | enum | 状态(valid/used/expired) |

## 6. 接口需求

### 6.1 用户认证接口
- POST /api/auth/register - 用户注册
- POST /api/auth/login - 用户登录
- POST /api/auth/logout - 用户登出
- POST /api/auth/forgot-password - 找回密码
- GET /api/user/profile - 获取个人信息
- PUT /api/user/profile - 更新个人信息

### 6.2 投稿管理接口
- POST /api/submissions - 创建投稿
- GET /api/submissions - 获取投稿列表
- GET /api/submissions/:id - 获取投稿详情
- DELETE /api/submissions/:id - 删除投稿

### 6.3 审稿接口
- GET /api/reviews/pending - 获取待审稿件
- POST /api/reviews/:submissionId - 提交审稿意见
- GET /api/reviews/history - 获取审稿历史

### 6.4 订单管理接口
- GET /api/tickets - 获取票务列表
- POST /api/orders - 创建订单
- GET /api/orders - 获取订单列表
- GET /api/orders/:id - 获取订单详情
- POST /api/orders/:id/pay - 支付订单
- GET /api/credentials/:orderId - 获取入会凭证

### 6.5 管理后台接口
- GET /api/admin/dashboard - 获取仪表盘数据
- GET /api/admin/submissions - 获取投稿列表
- GET /api/admin/orders - 获取订单列表
- GET /api/admin/users - 获取用户列表
- POST /api/admin/tickets - 创建票务
- PUT /api/admin/tickets/:id - 更新票务
