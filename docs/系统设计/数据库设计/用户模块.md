# 用户模块数据库设计

## 1. 模块概述

用户模块负责管理系统用户信息，包括投稿讲者、审稿人、参会者和管理员四种角色。用户信息包含账号、角色、状态等基本信息。

## 2. 表结构

### 2.1 用户表 (users)

```sql
CREATE TABLE `users` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `email` VARCHAR(100) NOT NULL COMMENT '邮箱',
  `password` VARCHAR(255) NOT NULL COMMENT '密码(BCrypt加密)',
  `name` VARCHAR(50) NOT NULL COMMENT '姓名',
  `role` TINYINT NOT NULL DEFAULT 3 COMMENT '角色:1-speaker,2-reviewer,3-attendee,4-admin',
  `status` TINYINT NOT NULL DEFAULT 1 COMMENT '状态:1-active,0-disabled',
  `avatar` VARCHAR(255) COMMENT '头像URL',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_email` (`email`),
  KEY `idx_role` (`role`),
  KEY `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

## 3. 字段说明

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键ID，自增 |
| email | VARCHAR(100) | 是 | 邮箱，唯一索引，作为登录账号 |
| password | VARCHAR(255) | 是 | 密码，使用BCrypt加密存储 |
| name | VARCHAR(50) | 是 | 用户真实姓名 |
| role | TINYINT | 是 | 用户角色，关联字典表 dict_code=user_role |
| status | TINYINT | 是 | 账号状态，关联字典表 dict_code=user_status |
| avatar | VARCHAR(255) | 否 | 用户头像URL |
| created_at | DATETIME | 是 | 账号创建时间 |
| updated_at | DATETIME | 是 | 账号最后更新时间 |

## 4. 索引说明

| 索引名 | 类型 | 字段 | 说明 |
|--------|------|------|------|
| PRIMARY | 主键 | id | 主键索引 |
| uk_email | 唯一 | email | 邮箱唯一性约束，用于登录 |
| idx_role | 普通 | role | 按角色查询用户 |
| idx_status | 普通 | status | 按状态查询用户 |

## 5. 数据字典

### 5.1 用户角色 (dict_code: user_role)

| item_code | item_value | item_name | 权限 |
|-----------|------------|-----------|------|
| speaker | 1 | 投稿讲者 | 投稿管理 |
| reviewer | 2 | 审稿人 | 审稿管理 |
| attendee | 3 | 参会者 | 订单管理、入会凭证 |
| admin | 4 | 管理员 | 所有权限 |

### 5.2 用户状态 (dict_code: user_status)

| item_code | item_value | item_name | 说明 |
|-----------|------------|-----------|------|
| active | 1 | 激活 | 账号正常可用 |
| disabled | 0 | 禁用 | 账号已被禁用，无法登录 |

## 6. 业务约束

1. **邮箱唯一性**：系统内邮箱必须唯一，作为用户唯一标识
2. **密码加密**：密码使用BCrypt加密，不存储明文
3. **角色限制**：用户角色只能从预定义的四种角色中选择
4. **状态控制**：只有status=1的用户可以登录系统

## 7. 关联模块

- **投稿模块**：speaker角色用户可以创建投稿
- **审稿模块**：reviewer角色用户可以审稿
- **订单模块**：所有用户都可以购买票务
- **凭证模块**：attendee角色用户可以使用入会凭证
