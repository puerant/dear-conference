# 字典模块数据库设计

## 1. 模块概述

字典模块是系统的基础配置模块，负责维护系统中所有的枚举类型数据。采用字典表（dictionary）和字典项表（dictionary_item）两级结构，实现统一的字典管理，提高系统的灵活性和可维护性。

遵循阿里巴巴Java开发手册（嵩山版）规范，使用TINYINT存储枚举值，不使用数据库层面的外键约束，所有外键关系由应用层代码维护。

## 2. 表结构

### 2.1 字典表 (dictionary)

```sql
CREATE TABLE `dictionary` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `dict_code` VARCHAR(50) NOT NULL COMMENT '字典编码，唯一',
  `dict_name` VARCHAR(100) NOT NULL COMMENT '字典名称',
  `description` VARCHAR(255) COMMENT '字典描述',
  `status` TINYINT NOT NULL DEFAULT 1 COMMENT '状态:1-启用,0-禁用',
  `sort_order` INT NOT NULL DEFAULT 0 COMMENT '排序',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_dict_code` (`dict_code`),
  KEY `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='字典表';
```

### 2.2 字典项表 (dictionary_item)

```sql
CREATE TABLE `dictionary_item` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `dict_id` BIGINT NOT NULL COMMENT '字典ID',
  `item_code` VARCHAR(50) NOT NULL COMMENT '字典项编码',
  `item_name` VARCHAR(100) NOT NULL COMMENT '字典项名称',
  `item_value` VARCHAR(50) NOT NULL COMMENT '字典项值',
  `description` VARCHAR(255) COMMENT '字典项描述',
  `status` TINYINT NOT NULL DEFAULT 1 COMMENT '状态:1-启用,0-禁用',
  `sort_order` INT NOT NULL DEFAULT 0 COMMENT '排序',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_dict_id` (`dict_id`),
  KEY `idx_item_code` (`item_code`),
  KEY `idx_status` (`status`),
  UNIQUE KEY `uk_dict_code_item_code` (`dict_id`, `item_code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='字典项表';
```

## 3. 字段说明

### 3.1 字典表字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键ID，自增 |
| dict_code | VARCHAR(50) | 是 | 字典编码，系统内唯一，如user_role、order_status |
| dict_name | VARCHAR(100) | 是 | 字典名称，如"用户角色"、"订单状态" |
| description | VARCHAR(255) | 否 | 字典描述，说明该字典的用途 |
| status | TINYINT | 是 | 字典状态，关联通用状态字典 |
| sort_order | INT | 是 | 排序号，用于字典列表展示 |
| created_at | DATETIME | 是 | 创建时间 |
| updated_at | DATETIME | 是 | 最后更新时间 |

### 3.2 字典项表字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键ID，自增 |
| dict_id | BIGINT | 是 | 所属字典ID，关联dictionary表 |
| item_code | VARCHAR(50) | 是 | 字典项编码，如speaker、paid |
| item_name | VARCHAR(100) | 是 | 字典项名称，如"投稿讲者"、"已支付" |
| item_value | VARCHAR(50) | 是 | 字典项值，存储到业务表的TINYINT值 |
| description | VARCHAR(255) | 否 | 字典项描述，说明该选项的含义 |
| status | TINYINT | 是 | 字典项状态，控制是否可选 |
| sort_order | INT | 是 | 排序号，用于下拉框展示顺序 |
| created_at | DATETIME | 是 | 创建时间 |
| updated_at | DATETIME | 是 | 最后更新时间 |

## 6. 业务约束

### 6.1 字典表约束
1. **编码唯一**：dict_code在系统内必须唯一
2. **命名规范**：dict_code采用小写+下划线命名，如user_role
3. **状态控制**：禁用的字典不可被业务模块使用

### 6.2 字典项表约束
1. **字典关联**：dict_id必须关联dictionary表中存在的字典
2. **编码唯一**：同一字典下item_code不能重复
3. **值唯一**：同一字典下item_value不能重复
4. **值类型**：item_value存储为字符串，实际业务表中存储为TINYINT
5. **排序控制**：sort_order影响下拉框展示顺序

### 6.3 使用规范
1. **业务表存储**：业务表使用TINYINT存储item_value的数值
2. **关联方式**：通过dict_code查询字典，再通过item_value获取item_name
3. **缓存策略**：字典数据变化频率低，建议缓存到Redis

## 7. 应用层使用示例

### 7.1 Java枚举定义

```java
// 用户角色枚举
public enum UserRole {
    SPEAKER(1, "speaker", "投稿讲者"),
    REVIEWER(2, "reviewer", "审稿人"),
    ATTENDEE(3, "attendee", "参会者"),
    ADMIN(4, "admin", "管理员");

    private final Integer value;
    private final String code;
    private final String name;
}
```

### 7.2 查询字典示例

```java
// 根据字典编码查询字典
Dictionary dictionary = dictionaryService.getByDictCode("user_role");

// 根据字典ID查询字典项列表
List<DictionaryItem> items = dictionaryItemService.getByDictId(dictionary.getId());

// 根据item_value获取item_name
String roleName = items.stream()
    .filter(item -> item.getItemValue().equals("1"))
    .findFirst()
    .map(DictionaryItem::getItemName)
    .orElse("");
```

### 7.3 外键维护示例

```java
// 保存用户时检查角色有效性
public void saveUser(User user) {
    Dictionary dictionary = dictionaryService.getByDictCode("user_role");
    boolean valid = dictionaryItemService.exists(
        dictionary.getId(), user.getRole()
    );
    if (!valid) {
        throw new BusinessException("用户角色无效");
    }
    userRepository.insert(user);
}
```

## 9. 关联模块

字典模块为以下模块提供枚举值维护：
- **用户模块**：user_role、user_status
- **投稿模块**：submission_status、review_result
- **票务模块**：ticket_status
- **订单模块**：order_status
- **凭证模块**：credential_status

所有模块的枚举字段存储item_value，展示时通过字典获取item_name。
