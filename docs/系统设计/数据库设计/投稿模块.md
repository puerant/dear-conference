# 投稿模块数据库设计

## 1. 模块概述

投稿模块负责管理学术会议的论文投稿流程，包括投稿信息提交、审稿人评审、审稿意见反馈等功能。该模块包含投稿表和审稿记录表。

## 2. 表结构

### 2.1 投稿表 (submissions)

```sql
CREATE TABLE `submissions` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `user_id` BIGINT NOT NULL COMMENT '讲者ID',
  `title` VARCHAR(200) NOT NULL COMMENT '投稿标题',
  `abstract` TEXT COMMENT '摘要',
  `category` VARCHAR(50) COMMENT '投稿分类',
  `file_url` VARCHAR(255) COMMENT '附件URL',
  `status` TINYINT NOT NULL DEFAULT 1 COMMENT '状态:1-pending,2-reviewing,3-accepted,4-rejected',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_status` (`status`),
  KEY `idx_category` (`category`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='投稿表';
```

### 2.2 审稿记录表 (reviews)

```sql
CREATE TABLE `reviews` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `submission_id` BIGINT NOT NULL COMMENT '投稿ID',
  `reviewer_id` BIGINT NOT NULL COMMENT '审稿人ID',
  `comment` TEXT COMMENT '审稿意见',
  `result` TINYINT COMMENT '审稿结果:1-accept,2-reject,3-revise',
  `score` INT COMMENT '评分(0-100)',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  PRIMARY KEY (`id`),
  KEY `idx_submission_id` (`submission_id`),
  KEY `idx_reviewer_id` (`reviewer_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='审稿记录表';
```

## 5. 数据字典

### 5.1 投稿状态 (dict_code: submission_status)

| item_code | item_value | item_name | 说明 |
|-----------|------------|-----------|------|
| pending | 1 | 待审稿 | 投稿已提交，等待分配审稿人 |
| reviewing | 2 | 审稿中 | 正在进行审稿评审 |
| accepted | 3 | 已录用 | 审稿通过，投稿被录用 |
| rejected | 4 | 已拒绝 | 审稿未通过，投稿被拒绝 |

### 5.2 审稿结果 (dict_code: review_result)

| item_code | item_value | item_name | 说明 |
|-----------|------------|-----------|------|
| accept | 1 | 录用 | 审稿通过，建议录用 |
| reject | 2 | 拒绝 | 审稿不通过，建议拒绝 |
| revise | 3 | 需修改 | 需要修改后重新提交 |

## 6. 业务约束

### 6.1 投稿表约束
1. **讲者关联**：user_id必须关联users表中存在的用户，且用户角色为speaker
2. **状态流转**：投稿状态流转为 pending → reviewing → accepted/rejected
3. **附件限制**：file_url存储论文附件，文件大小限制10MB

### 6.2 审稿记录表约束
1. **投稿关联**：submission_id必须关联submissions表中存在的投稿
2. **审稿人关联**：reviewer_id必须关联users表中存在的用户，且用户角色为reviewer
3. **唯一审稿**：同一审稿人对同一投稿只能审稿一次
4. **评分范围**：score取值范围为0-100

## 7. 业务流程

### 7.1 投稿流程
```
讲者提交投稿
    ↓
状态: pending (待审稿)
    ↓
管理员分配审稿人
    ↓
状态: reviewing (审稿中)
    ↓
审稿人提交审稿意见
    ↓
状态: accepted/rejected (已录用/已拒绝)
```

### 7.2 审稿流程
1. 审稿人查看待审稿列表
2. 选择投稿进入审稿页面
3. 阅读投稿内容和附件
4. 填写审稿意见、评分和审稿结果
5. 提交审稿记录

## 8. 关联模块

- **用户模块**：投稿关联讲者，审稿关联审稿人
- **字典模块**：使用字典表维护投稿状态和审稿结果
