# 订单模块数据库设计

## 1. 模块概述

订单模块负责管理用户的购票订单，包括订单创建、支付、取消、退款等流程。订单模块是连接用户、票务和凭证的核心模块。

## 2. 表结构

### 2.1 订单表 (orders)

```sql
CREATE TABLE `orders` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `order_no` VARCHAR(32) NOT NULL COMMENT '订单号',
  `user_id` BIGINT NOT NULL COMMENT '用户ID',
  `ticket_id` BIGINT NOT NULL COMMENT '票务ID',
  `quantity` INT NOT NULL COMMENT '数量',
  `total_amount` DECIMAL(10, 2) NOT NULL COMMENT '总金额',
  `status` TINYINT NOT NULL DEFAULT 1 COMMENT '状态:1-unpaid,2-paid,3-cancelled,4-refunded',
  `attendee_name` VARCHAR(50) COMMENT '参会人姓名',
  `attendee_email` VARCHAR(100) COMMENT '参会人邮箱',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `paid_at` DATETIME COMMENT '支付时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_order_no` (`order_no`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_ticket_id` (`ticket_id`),
  KEY `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='订单表';
```

## 3. 字段说明

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键ID，自增 |
| order_no | VARCHAR(32) | 是 | 订单号，唯一标识，格式：ORD+时间戳+随机数 |
| user_id | BIGINT | 是 | 下单用户ID，关联users表 |
| ticket_id | BIGINT | 是 | 购买的票务ID，关联tickets表 |
| quantity | INT | 是 | 购票数量，必须>=1 |
| total_amount | DECIMAL(10,2) | 是 | 订单总金额，= ticket.price * quantity |
| status | TINYINT | 是 | 订单状态，关联字典表 dict_code=order_status |
| attendee_name | VARCHAR(50) | 否 | 参会人真实姓名 |
| attendee_email | VARCHAR(100) | 否 | 参会人联系邮箱 |
| created_at | DATETIME | 是 | 订单创建时间 |
| paid_at | DATETIME | 否 | 订单支付完成时间 |

## 4. 索引说明

| 索引名 | 类型 | 字段 | 说明 |
|--------|------|------|------|
| PRIMARY | 主键 | id | 主键索引 |
| uk_order_no | 唯一 | order_no | 订单号唯一性约束 |
| idx_user_id | 普通 | user_id | 查询用户的订单列表 |
| idx_ticket_id | 普通 | ticket_id | 查询票务的订单 |
| idx_status | 普通 | status | 按状态筛选订单 |

## 5. 数据字典

### 5.1 订单状态 (dict_code: order_status)

| item_code | item_value | item_name | 说明 |
|-----------|------------|-----------|------|
| unpaid | 1 | 未支付 | 订单已创建，等待支付 |
| paid | 2 | 已支付 | 订单已支付完成，等待生成凭证 |
| cancelled | 3 | 已取消 | 订单未支付时取消 |
| refunded | 4 | 已退款 | 订单支付后申请退款成功 |

## 6. 业务约束

### 6.1 订单创建约束
1. **用户关联**：user_id必须关联users表中存在的用户
2. **票务关联**：ticket_id必须关联tickets表中存在的票务
3. **票务状态**：只能购买status=1（上架）的票务
4. **库存检查**：购买数量不能超过票务可用库存 (stock - sold_count)
5. **数量限制**：quantity必须 >= 1

### 6.2 订单状态流转约束
```
unpaid (未支付)
    ├─→ paid (已支付)
    │        └─→ refunded (已退款)
    └─→ cancelled (已取消)
```

### 6.3 金额计算规则
- **total_amount** = ticket.price × quantity
- total_amount保留2位小数

### 6.4 参会人信息
- 支付前可修改 attendee_name 和 attendee_email
- 支付后参会人信息不可修改
- 参会人信息用于生成入会凭证

## 7. 业务流程

### 7.1 下单流程
```
用户选择票务
    ↓
填写参会人信息
    ↓
检查票务库存和状态
    ↓
创建订单 (status: unpaid)
    ↓
生成订单号 order_no
    ↓
跳转支付页面
```

### 7.2 支付流程
```
用户完成支付
    ↓
更新订单状态 status: paid
    ↓
记录支付时间 paid_at
    ↓
更新票务已售数量 sold_count += quantity
    ↓
生成入会凭证
```

### 7.3 取消订单流程
```
用户发起取消
    ↓
检查订单状态 (必须为unpaid)
    ↓
更新订单状态 status: cancelled
    ↓
订单完成，无需其他操作
```

### 7.4 退款流程
```
用户发起退款
    ↓
检查订单状态 (必须为paid)
    ↓
管理员审核退款
    ↓
更新订单状态 status: refunded
    ↓
更新票务已售数量 sold_count -= quantity
    ↓
取消入会凭证状态 status: expired
```

## 8. 订单号生成规则

格式：`ORD{timestamp}{random}`

示例：`ORD20260225143000123456`

- ORD：订单前缀
- 20260225143000：时间戳 (年月日时分秒)
- 123456：6位随机数

## 9. 关联模块

- **用户模块**：订单关联下单用户
- **票务模块**：订单关联购买的票务
- **凭证模块**：订单支付后生成入会凭证
- **字典模块**：使用字典表维护订单状态
