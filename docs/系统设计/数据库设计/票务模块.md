# 票务模块数据库设计

## 1. 模块概述

票务模块负责管理会议的票务信息，包括不同类型的票务（如早鸟票、普通票、VIP票等）、价格、库存管理等功能。票务信息为订单模块提供基础数据支持。

## 2. 表结构

### 2.1 票务表 (tickets)

```sql
CREATE TABLE `tickets` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `name` VARCHAR(100) NOT NULL COMMENT '票务名称',
  `price` DECIMAL(10, 2) NOT NULL COMMENT '价格',
  `description` TEXT COMMENT '描述',
  `stock` INT NOT NULL DEFAULT 0 COMMENT '库存',
  `sold_count` INT NOT NULL DEFAULT 0 COMMENT '已售数量',
  `status` TINYINT NOT NULL DEFAULT 1 COMMENT '状态:1-active,0-disabled',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='票务表';
```

## 3. 字段说明

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键ID，自增 |
| name | VARCHAR(100) | 是 | 票务名称，如"早鸟票"、"普通票"等 |
| price | DECIMAL(10,2) | 是 | 票务价格，单位：元 |
| description | TEXT | 否 | 票务描述，说明包含的权益 |
| stock | INT | 是 | 票务库存数量 |
| sold_count | INT | 是 | 已售出数量 |
| status | TINYINT | 是 | 票务状态，关联字典表 dict_code=ticket_status |
| created_at | DATETIME | 是 | 票务创建时间 |
| updated_at | DATETIME | 是 | 票务最后更新时间 |

## 4. 索引说明

| 索引名 | 类型 | 字段 | 说明 |
|--------|------|------|------|
| PRIMARY | 主键 | id | 主键索引 |
| idx_status | 普通 | status | 查询有效的票务 |

## 5. 数据字典

### 5.1 票务状态 (dict_code: ticket_status)

| item_code | item_value | item_name | 说明 |
|-----------|------------|-----------|------|
| active | 1 | 上架 | 票务正在销售中，用户可以购买 |
| disabled | 0 | 下架 | 票务已下架，用户无法购买 |

## 6. 业务约束

1. **库存管理**：
   - sold_count不能超过stock
   - 库存不足时禁止下单
   - 下单成功后sold_count+1，取消订单后sold_count-1

2. **价格控制**：
   - price必须大于0
   - 价格精度为小数点后2位

3. **状态控制**：
   - 只有status=1的票务可以购买
   - 票务下架前需确保无未完成订单

4. **库存补充**：
   - 管理员可以增加stock数量
   - stock更新后不影响已售数量

## 7. 业务规则

### 7.1 票务类型示例
| 票务名称 | 价格 | 库存 | 描述 |
|---------|------|------|------|
| 早鸟票 | 299.00 | 500 | 优惠票，限前500名 |
| 普通票 | 499.00 | 1000 | 标准参会票 |
| VIP票 | 999.00 | 100 | 包含VIP晚宴和前排座位 |

### 7.2 库存管理规则
1. 创建票务时设定初始库存stock
2. 用户下单时检查：stock - sold_count >= quantity
3. 订单支付成功：sold_count += quantity
4. 订单取消：sold_count -= quantity

## 8. 关联模块

- **订单模块**：订单表关联票务表，一个票务可对应多个订单
- **字典模块**：使用字典表维护票务状态
