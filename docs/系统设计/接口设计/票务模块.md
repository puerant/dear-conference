# 票务模块分析

## 1. 模块概述

票务模块负责管理会议的票务类型、价格、库存等信息。该模块为订单模块提供票务数据基础，管理员可以通过该模块管理会议的票务产品。

## 2. 功能模块划分

### 2.1 票务查询（所有用户）
- 查看票务列表
- 查看票务详情
- 查看可用票务（库存>0且状态为上架）

### 2.2 票务管理（管理员）
- 创建票务
- 修改票务信息
- 删除票务
- 修改票务状态（上架/下架）
- 增加库存
- 查看票务统计

### 2.3 票务分类（可选）
- 按类型分类：早鸟票、普通票、VIP票
- 按时段分类：早鸟期、正常期、现场票

## 4. API实现逻辑

### 4.1 查询票务列表逻辑

```
1. 接收参数：page, pageSize, status(可选)
2. 构建查询条件：
   - WHERE 1=1
   - AND status = ? (status非空时)
3. 分页查询：
   - SELECT * FROM tickets WHERE ... ORDER BY sort_order, created_at DESC LIMIT ?, ?
4. 计算可用库存：
   - available = stock - sold_count
5. 返回分页结果
```

### 4.2 查询票务详情逻辑

```
1. 接收参数：ticketId
2. 查询票务：
   - SELECT * FROM tickets WHERE id = ?
3. 票务不存在则抛出异常
4. 计算可用库存：
   - available = stock - sold_count
5. 返回票务详情（包含可用库存）
```

### 4.3 查询可购买票务逻辑

```
1. 接收参数：page, pageSize
2. 查询可购买票务：
   - SELECT * FROM tickets
     WHERE status = 1 AND stock > sold_count
     ORDER BY sort_order, created_at DESC
     LIMIT ?, ?
3. 计算每个票务的可用库存
4. 返回结果
```

### 4.4 创建票务逻辑

```
1. 接收参数：name, price, description, stock, sortOrder
2. 权限校验：
   - 提取Token，获取user_id
   - 用户角色必须为admin
3. 参数校验：
   - name非空，长度1-100
   - price > 0，保留2位小数
   - stock >= 0
   - description长度0-1000
4. 创建票务：
   - INSERT INTO tickets (name, price, description, stock, sold_count, status, sort_order)
   - sold_count默认为0
   - status默认为1（上架）
5. 返回票务ID
```

### 4.5 修改票务信息逻辑

```
1. 接收参数：ticketId, name, price, description, sortOrder
2. 权限校验：必须为admin角色
3. 查询票务，验证存在
4. 构建更新字段（只更新提供的参数）
5. 更新票务：
   - UPDATE tickets SET ... WHERE id = ?
6. 返回更新成功
```

### 4.6 修改票务状态逻辑

```
1. 接收参数：ticketId, status
2. 权限校验：必须为admin角色
3. 查询票务，验证存在
4. 检查：
   - 下架前确认无未完成订单
5. 更新状态：
   - UPDATE tickets SET status = ?, updated_at = NOW() WHERE id = ?
6. 返回更新成功
```

### 4.7 调整库存逻辑

```
1. 接收参数：ticketId, stockChange, operation(add/reduce)
2. 权限校验：必须为admin角色
3. 查询票务
4. 验证：
   - stock + stockChange >= sold_count (减少库存时)
   - stock + stockChange >= 0 (库存不能为负)
5. 更新库存：
   - UPDATE tickets SET stock = stock + ? WHERE id = ?
6. 记录库存变更日志（如需要）
7. 返回更新成功
```

### 4.8 删除票务逻辑

```
1. 接收参数：ticketId
2. 权限校验：必须为admin角色
3. 查询票务
4. 验证：
   - 票务存在
   - sold_count为0（已售出的票务不能删除）
5. 删除票务：
   - DELETE FROM tickets WHERE id = ?
6. 返回删除成功
```

### 4.9 票务统计逻辑

```
1. 权限校验：必须为admin角色
2. 查询统计信息：
   - 总票务数：SELECT count(*) FROM tickets
   - 上架票务数：SELECT count(*) FROM tickets WHERE status = 1
   - 总库存：SELECT sum(stock) FROM tickets
   - 已售数量：SELECT sum(sold_count) FROM tickets
   - 总销售额：SELECT sum(price * sold_count) FROM tickets
3. 返回统计结果
```

### 5.2 数据流向

```
票务创建流程：
管理员 → 创建票务接口 → tickets表 → 记录初始库存

票务购买流程：
用户 → 查看可购买票务 → 选择票务 → 创建订单 → 订单表记录ticket_id

库存更新流程：
订单支付成功 → 更新sold_count → 检查库存是否充足

库存补充流程：
管理员 → 调整库存接口 → 更新stock → 增加可用库存
```

## 6. 业务规则

### 6.1 票务创建规则
1. 票务名称必填，长度1-100字符
2. 价格必须大于0，保留2位小数
3. 库存必须 >= 0
4. 描述选填，最多1000字符
5. 默认状态为上架
6. 默认已售数量为0

### 6.2 票务修改规则
1. 只有管理员可以修改票务
2. 已售出的票务（sold_count > 0）不能删除
3. 下架前需确认无未完成订单
4. 库存减少不能低于已售数量

### 6.3 库存管理规则
1. 可用库存 = stock - sold_count
2. 下单时检查：stock - sold_count >= quantity
3. 支付成功：sold_count += quantity
4. 取消订单：sold_count -= quantity
5. 退款：sold_count -= quantity
6. 管理员可以动态调整stock数量

### 6.4 票务展示规则
1. 只展示status=1（上架）的票务
2. 按sort_order排序
3. 同时创建时间倒序
4. 显示可用库存（stock - sold_count）

## 7. 异常处理

| 异常码 | 异常信息 | 说明 |
|--------|---------|------|
| 3001 | 票务不存在 | 查询的票务不存在 |
| 3002 | 票务名称过长 | 超过100字符限制 |
| 3003 | 票务价格无效 | 价格必须大于0 |
| 3004 | 库存不足 | 购买数量超过可用库存 |
| 3005 | 票务已售出不能删除 | sold_count > 0 |
| 3006 | 库存调整无效 | 调整后库存低于已售数量 |
| 3007 | 票务有未完成订单不能下架 | 存在未支付或支付中的订单 |

## 9. 数据库操作

### 9.1 主要查询语句

```sql
-- 查询票务列表
SELECT * FROM tickets
WHERE 1=1
  AND status = ?
ORDER BY sort_order ASC, created_at DESC
LIMIT ?, ?;

-- 查询可购买票务
SELECT *,
       (stock - sold_count) as available
FROM tickets
WHERE status = 1
  AND stock > sold_count
ORDER BY sort_order ASC, created_at DESC
LIMIT ?, ?;

-- 查询票务详情
SELECT * FROM tickets WHERE id = ?;

-- 创建票务
INSERT INTO tickets (name, price, description, stock, sold_count, status, sort_order)
VALUES (?, ?, ?, ?, 0, 1, ?);

-- 修改票务信息
UPDATE tickets
SET name = ?, price = ?, description = ?, sort_order = ?, updated_at = NOW()
WHERE id = ?;

-- 修改票务状态
UPDATE tickets
SET status = ?, updated_at = NOW()
WHERE id = ?;

-- 调整库存
UPDATE tickets
SET stock = stock + ?, updated_at = NOW()
WHERE id = ?;

-- 增加已售数量
UPDATE tickets
SET sold_count = sold_count + ?, updated_at = NOW()
WHERE id = ?;

-- 减少已售数量
UPDATE tickets
SET sold_count = sold_count - ?, updated_at = NOW()
WHERE id = ?;

-- 删除票务
DELETE FROM tickets WHERE id = ?;

-- 票务统计
SELECT
    count(*) as total_count,
    sum(CASE WHEN status = 1 THEN 1 ELSE 0 END) as active_count,
    sum(stock) as total_stock,
    sum(sold_count) as total_sold,
    sum(price * sold_count) as total_sales
FROM tickets;
```

### 9.2 外键维护（应用层）

```java
// 检查票务是否存在
public boolean ticketExists(Long ticketId) {
    return ticketMapper.selectCountById(ticketId) > 0;
}

// 检查库存是否充足
public boolean checkStock(Long ticketId, Integer quantity) {
    Ticket ticket = ticketMapper.selectById(ticketId);
    return (ticket.getStock() - ticket.getSoldCount()) >= quantity;
}

// 更新已售数量
public void increaseSoldCount(Long ticketId, Integer quantity) {
    ticketMapper.increaseSoldCount(ticketId, quantity);
    // 清除缓存
    redisTemplate.delete("ticket:" + ticketId);
}

// 减少已售数量
public void decreaseSoldCount(Long ticketId, Integer quantity) {
    ticketMapper.decreaseSoldCount(ticketId, quantity);
    // 清除缓存
    redisTemplate.delete("ticket:" + ticketId);
}
```
