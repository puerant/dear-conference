# 凭证模块分析

## 1. 模块概述

凭证模块负责管理参会者的入会凭证，包括凭证生成、二维码生成、座位分配、凭证核销（使用）等功能。凭证是订单支付成功后自动生成的入会证明，是参会者进入会议的必要凭证。

## 2. 功能模块划分

### 2.1 凭证查询（参会者）
- 查看我的凭证列表
- 查看凭证详情
- 下载凭证（PDF或图片）
- 发送凭证到邮箱

### 2.2 凭证管理（管理员）
- 查询所有凭证
- 查询凭证详情
- 查询凭证使用记录
- 凭证统计
- 手动生成凭证

### 2.3 凭证核销（现场工作人员）
- 扫码核销
- 手动核销（输入凭证号）
- 查询核销记录
- 核销统计

## 4. API实现逻辑

### 4.1 查询凭证列表逻辑

```
1. 接收参数：page, pageSize, status(可选)
2. 权限校验：提取Token，获取user_id
3. 查询订单：
   - SELECT id FROM orders WHERE user_id = ?
4. 查询凭证：
   - SELECT c.*, o.ticket_id, o.order_no
     FROM credentials c
     LEFT JOIN orders o ON c.order_id = o.id
     WHERE o.user_id = ?
       AND c.status = ? (status非空时)
     ORDER BY c.created_at DESC
     LIMIT ?, ?
5. 关联查询：
   - 关联tickets表获取票务信息
   - 关联dictionary表获取状态名称
6. 返回分页结果
```

### 4.2 查询凭证详情逻辑

```
1. 接收参数：credentialNo
2. 权限校验：提取Token，获取user_id
3. 查询凭证：
   - SELECT c.*, o.order_no, o.attendee_name, o.attendee_email,
            t.name as ticket_name, t.price as ticket_price
     FROM credentials c
     LEFT JOIN orders o ON c.order_id = o.id
     LEFT JOIN tickets t ON o.ticket_id = t.id
     WHERE c.credential_no = ?
4. 验证权限：
   - 普通用户：只能查询自己的凭证
   - 管理员/工作人员：可以查询所有凭证
5. 返回凭证详情（包含二维码数据）
```

### 4.3 生成凭证逻辑

```
1. 接收参数：orderId
2. 查询订单信息：
   - SELECT * FROM orders WHERE id = ?
3. 验证：
   - 订单存在
   - 订单状态为2（已支付）
4. 查询是否已生成凭证：
   - SELECT count FROM credentials WHERE order_id = ?
5. 已生成则不重复生成
6. 根据订单quantity循环生成凭证：
   FOR i FROM 1 TO quantity DO
     a. 生成凭证号：CRD{timestamp}{random}
     b. 分配座位信息：调用座位分配算法
     c. 生成二维码数据：JSON格式
     d. 插入凭证：INSERT INTO credentials (order_id, credential_no, qr_code, seat_info)
7. 返回生成的凭证列表
```

### 4.4 手动生成凭证逻辑

```
1. 权限校验：必须为admin角色
2. 接收参数：orderId
3. 调用生成凭证逻辑
4. 返回生成结果
```

### 4.5 扫码核销逻辑

```
1. 接收参数：qrCodeData
2. 权限校验：工作人员Token
3. 解析二维码数据获取credentialNo
4. 查询凭证：
   - SELECT * FROM credentials WHERE credential_no = ?
5. 验证：
   - 凭证存在
   - status为1（有效）
   - 验证时效（检查会议是否结束）
6. 更新凭证状态：
   - UPDATE credentials SET status = 2, used_at = NOW() WHERE id = ?
7. 记录核销信息：
   - INSERT INTO checkin_records (credential_no, checkin_time, operator, location)
8. 返回核销成功（包含参会人信息）
```

### 4.6 手动核销逻辑

```
1. 接收参数：credentialNo
2. 权限校验：工作人员Token
3. 查询凭证
4. 验证：
   - 凭证存在
   - status为1（有效）
5. 更新凭证状态：
   - UPDATE credentials SET status = 2, used_at = NOW() WHERE id = ?
6. 记录核销信息
7. 返回核销成功
```

### 4.7 验证凭证逻辑

```
1. 接收参数：credentialNo
2. 权限校验：工作人员Token
3. 查询凭证
4. 关联查询订单和票务信息
5. 返回凭证信息（包含状态、是否可用）
```

### 4.8 查询核销记录逻辑

```
1. 接收参数：page, pageSize, date(可选)
2. 权限校验：工作人员Token
3. 构建查询条件：
   - WHERE 1=1
   - AND DATE(checkin_time) = ? (date非空时)
4. 分页查询：
   - SELECT cr.*, c.credential_no, o.attendee_name
     FROM checkin_records cr
     LEFT JOIN credentials c ON cr.credential_no = c.credential_no
     LEFT JOIN orders o ON c.order_id = o.id
     ORDER BY cr.checkin_time DESC
     LIMIT ?, ?
5. 返回分页结果
```

### 4.9 凭证统计逻辑

```
1. 权限校验：必须为admin角色
2. 查询统计信息：
   - 总凭证数：SELECT count(*) FROM credentials
   - 有效凭证数：SELECT count(*) FROM credentials WHERE status = 1
   - 已使用凭证数：SELECT count(*) FROM credentials WHERE status = 2
   - 已过期凭证数：SELECT count(*) FROM credentials WHERE status = 3
   - 今日核销数：SELECT count(*) FROM checkin_records WHERE DATE(checkin_time) = CURDATE()
3. 返回统计结果
```

### 4.10 凭证失效逻辑

```
1. 接收参数：orderId
2. 查询订单关联的所有凭证
   - SELECT * FROM credentials WHERE order_id = ?
3. 遍历凭证，更新状态：
   - UPDATE credentials SET status = 3 WHERE id IN (...)
4. 返回失效成功
```

### 5.2 数据流向

```
凭证生成流程：
订单支付成功 → 生成凭证接口 → credentials表 → 生成二维码 → 发送邮件

凭证核销流程：
参会者出示二维码 → 工作人员扫码 → 验证凭证 → 更新状态 → 记录核销

凭证失效流程：
订单退款 → 失效凭证接口 → 更新凭证状态 → 凭证不可用
```

## 6. 业务规则

### 6.1 凭证生成规则
1. 订单支付成功后自动生成
2. 凭证数量 = 订单的quantity数量
3. 凭证号在系统内唯一
4. 凭证号格式：CRD{timestamp}{random}
5. 默认状态为1（有效）
6. 自动分配座位信息

### 6.2 凭证核销规则
1. 只有status=1（有效）的凭证可以核销
2. 凭证核销为一次性操作
3. 核销时记录使用时间used_at
4. 记录核销操作人和核销位置
5. 核销成功后不可重复核销

### 6.3 座位分配规则
1. 普通票：按A、B、C区域顺序分配
2. VIP票：分配VIP区域前排座位
3. 早鸟票：优先分配前排座位
4. 同一订单多个凭证分配相邻座位
5. 座位信息格式：{区域}区{排}排{座}座

### 6.4 凭证失效规则
1. 订单退款后凭证自动失效
2. 失效凭证状态更新为3
3. 失效凭证无法核销

### 6.5 凭证下载规则
1. 只有凭证创建者可以下载
2. 支持PDF和图片格式
3. 下载内容包含凭证信息和二维码

## 7. 异常处理

| 异常码 | 异常信息 | 说明 |
|--------|---------|------|
| 5001 | 凭证不存在 | 查询的凭证不存在 |
| 5002 | 凭证已失效 | 凭证状态不是有效 |
| 5003 | 凭证已核销 | 凭证已被使用 |
| 5004 | 凭证已过期 | 凭证已失效或会议已结束 |
| 5005 | 无权操作此凭证 | 不是凭证的拥有者 |
| 5006 | 二维码无效 | 二维码数据格式错误 |
| 5007 | 订单未支付 | 未支付订单不能生成凭证 |
| 5008 | 凭证已存在 | 订单已生成凭证 |

```sql
       t.name as ticket_name, t.price, t.description as ticket_desc

LEFT JOIN orders o ON c.order_id = o.id
LEFT JOIN tickets t ON o.ticket_id = t.id
WHERE c.credential_no = ?;

-- 查询凭证关联订单
SELECT * FROM credentials WHERE order_id = ?;

-- 生成凭证
INSERT INTO credentials (order_id, credential_no, qr_code, seat_info, status)
VALUES (?, ?, ?, ?, 1);

-- 核销凭证
UPDATE credentials
SET status = 2, used_at = NOW()
WHERE credential_no = ?;

-- 失效凭证
UPDATE credentials
SET status = 3
WHERE order_id = ?;

-- 凭证统计
SELECT
    count(*) as total_count,
    sum(CASE WHEN status = 1 THEN 1 ELSE 0 END) as valid_count,
    sum(CASE WHEN status = 2 THEN 1 ELSE 0 END) as used_count,
    sum(CASE WHEN status = 3 THEN 1 ELSE 0 END) as expired_count
FROM credentials;

-- 今日核销统计
SELECT count(*) as today_checkin
FROM checkin_records
WHERE DATE(checkin_time) = CURDATE();
```

```java
// 检查凭证是否存在
public boolean credentialExists(String credentialNo) {
    return credentialMapper.selectCountByCredentialNo(credentialNo) > 0;
}

// 检查凭证是否有效
public boolean isValid(String credentialNo) {
    Credential credential = credentialMapper.selectByCredentialNo(credentialNo);
    return credential != null && credential.getStatus() == 1;
}

// 检查凭证是否已使用
public boolean isUsed(String credentialNo) {
    Credential credential = credentialMapper.selectByCredentialNo(credentialNo);
    return credential != null && credential.getStatus() == 2;
}

// 批量生成凭证
@Transactional
public void generateCredentials(Order order) {
    // 检查是否已生成
    List<Credential> existing = credentialMapper.selectByOrderId(order.getId());
    if (!existing.isEmpty()) {
        return;
    }

    // 生成凭证
    for (int i = 0; i < order.getQuantity(); i++) {
        String credentialNo = generateCredentialNo();
        String seatInfo = seatService.allocateSeat(order.getTicketId());
        String qrCode = buildQrCode(credentialNo, order, seatInfo);

        Credential credential = new Credential();
        credential.setOrderId(order.getId());
        credential.setCredentialNo(credentialNo);
        credential.setQrCode(qrCode);
        credential.setSeatInfo(seatInfo);
        credential.setStatus(1);

        credentialMapper.insert(credential);
    }
}
```

## 11. 座位分配算法

### 11.1 座位分配规则

```
区域划分：
- A区：普通座位，1-20排，每排50座
- B区：普通座位，1-20排，每排50座
- C区：普通座位，1-20排，每排50座
- VIP区：VIP座位，1-5排，每排20座

分配策略：
1. 普通票：优先A区 → B区 → C区
2. VIP票：固定VIP区
3. 早鸟票：优先A区前排
4. 同订单多凭证：分配相邻座位
```

### 11.2 座位分配代码

```java
public String allocateSeat(Long ticketId, Integer quantity) {
    Ticket ticket = ticketMapper.selectById(ticketId);

    // VIP票分配VIP区
    if (ticket.getName().contains("VIP")) {
        return allocateVipSeats(quantity);
    }

    // 早鸟票优先前排
    boolean preferFront = ticket.getName().contains("早鸟");

    return allocateNormalSeats(quantity, preferFront);
}
```
