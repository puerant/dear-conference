# 字典模块分析

## 1. 模块概述

字典模块是系统的基础配置模块，负责维护系统中所有的枚举类型数据。采用字典表（dictionary）和字典项表（dictionary_item）两级结构，实现统一的字典管理。业务表中使用TINYINT存储枚举值，字典表负责维护item_code（编码）、item_name（名称）和item_value（数值）的映射关系。

遵循阿里巴巴Java开发手册（嵩山版）规范，不使用数据库层面的外键约束，所有外键关系由应用层代码维护。

## 2. 功能模块划分

### 2.1 字典查询（所有模块）
- 查询字典列表
- 查询字典详情

- 根据item_value查询item_name

### 2.2 字典管理（管理员）

### 2.3 字典项管理（管理员）

- 删除字典项
- 修改字典项状态（启用/禁用）
- 调整字典项排序

### 2.4 字典缓存
- 启动时加载字典到缓存
- 字典变更时刷新缓存
- 定时刷新缓存

## 4. API实现逻辑

### 4.1 查询字典列表逻辑

```
1. 接收参数：page, pageSize, status(可选)
2. 构建查询条件：
   - WHERE 1=1
   - AND status = ? (status非空时)
3. 分页查询：
   - SELECT * FROM dictionary WHERE ... ORDER BY sort_order ASC LIMIT ?, ?
4. 返回分页结果
```

### 4.2 查询字典详情逻辑

```
1. 接收参数：dictCode
2. 查询字典：
   - SELECT * FROM dictionary WHERE dict_code = ?
3. 字典不存在则抛出异常
4. 查询字典项列表：
   - SELECT * FROM dictionary_item WHERE dict_id = ? AND status = 1 ORDER BY sort_order ASC
5. 返回字典详情（包含字典项）
```

### 4.3 查询字典项列表逻辑

```
1. 接收参数：dictCode, status(可选)
2. 查询字典获取dict_id
3. 查询字典项：
   - SELECT * FROM dictionary_item WHERE dict_id = ? AND status = ? ORDER BY sort_order ASC
4. 返回字典项列表
```

### 4.4 根据item_value查询item_name逻辑

```
1. 接收参数：dictCode, itemValue
2. 查询字典获取dict_id
3. 查询字典项：
   - SELECT item_name FROM dictionary_item WHERE dict_id = ? AND item_value = ? AND status = 1
4. 返回item_name
```

### 4.5 创建字典逻辑

```
1. 权限校验：必须为admin角色
2. 接收参数：dictCode, dictName, description, sortOrder
3. 参数校验：
   - dictCode非空，1-50字符
   - dictName非空，1-100字符
   - dictCode格式：小写字母+下划线
4. 检查dictCode是否已存在：
   - SELECT count FROM dictionary WHERE dict_code = ?
5. 已存在则抛出异常
6. 创建字典：
   - INSERT INTO dictionary (dict_code, dict_name, description, status, sort_order)
   - status默认为1（启用）
7. 清除字典列表缓存
8. 返回字典ID
```

### 4.6 修改字典信息逻辑

```
1. 权限校验：必须为admin角色
2. 接收参数：dictionaryId, dictName, description, sortOrder
3. 查询字典，验证存在
4. 构建更新字段（只更新提供的参数）
5. 更新字典：
   - UPDATE dictionary SET ... WHERE id = ?
6. 清除字典列表缓存
7. 返回更新成功
```

### 4.7 修改字典状态逻辑

```
1. 权限校验：必须为admin角色
2. 接收参数：dictionaryId, status
3. 查询字典，验证存在
4. 更新状态：
   - UPDATE dictionary SET status = ?, updated_at = NOW() WHERE id = ?
5. 清除字典列表缓存
6. 返回更新成功
```

### 4.8 删除字典逻辑

```
1. 权限校验：必须为admin角色
2. 接收参数：dictionaryId
3. 查询字典
4. 检查：
   - 系统预置字典不允许删除
5. 删除字典项：
   - DELETE FROM dictionary_item WHERE dict_id = ?
6. 删除字典：
   - DELETE FROM dictionary WHERE id = ?
7. 清除字典列表缓存
8. 返回删除成功
```

### 4.9 创建字典项逻辑

```
1. 权限校验：必须为admin角色
2. 接收参数：dictId, itemCode, itemName, itemValue, sortOrder
3. 查询字典，验证存在
4. 参数校验：
   - itemCode非空，1-50字符
   - itemName非空，1-100字符
   - itemValue非空，1-50字符
5. 检查itemCode是否重复：
   - SELECT count FROM dictionary_item WHERE dict_id = ? AND item_code = ?
6. 检查itemValue是否重复：
   - SELECT count FROM dictionary_item WHERE dict_id = ? AND item_value = ?
7. 插入字典项：
   - INSERT INTO dictionary_item (dict_id, item_code, item_name, item_value, status, sort_order)
   - status默认为1（启用）
8. 清除该字典的缓存
9. 返回字典项ID
```

### 4.10 修改字典项逻辑

```
1. 权限校验：必须为admin角色
2. 接收参数：itemId, itemName, itemValue, sortOrder
3. 查询字典项，验证存在
4. 构建更新字段（只更新提供的参数）
5. 更新字典项：
   - UPDATE dictionary_item SET ... WHERE id = ?
6. 清除该字典的缓存
7. 返回更新成功
```

### 4.11 修改字典项状态逻辑

```
1. 权限校验：必须为admin角色
2. 接收参数：itemId, status
3. 查询字典项，验证存在
4. 更新状态：
   - UPDATE dictionary_item SET status = ?, updated_at = NOW() WHERE id = ?
5. 清除该字典的缓存
6. 返回更新成功
```

### 4.12 批量调整排序逻辑

```
1. 权限校验：必须为admin角色
2. 接收参数：dictId, items [{id, sortOrder}]
3. 遍历items：
   - UPDATE dictionary_item SET sort_order = ? WHERE id = ?
4. 清除该字典的缓存
5. 返回更新成功
```

### 4.13 重新加载缓存逻辑

```
1. 权限校验：必须为admin角色
2. 查询所有字典：
   - SELECT * FROM dictionary WHERE status = 1
3. 遍历字典：
   - 查询所有启用的字典项
   - 存储到Redis：dict:{dictCode} -> List<DictionaryItem>
4. 返回加载成功
```

### 4.14 启动时加载字典逻辑

```
1. 系统启动时自动执行
2. 查询所有启用的字典和字典项
3. 加载到Redis缓存：
   - dict:list -> List<Dictionary>
   - dict:{dictCode} -> List<DictionaryItem>
   - dict:map -> Map<dictCode, Map<itemValue, itemName>>
4. 日志记录加载的字典数量
```

### 5.2 数据流向

```
字典查询流程：
业务模块 → 根据dictCode查询 → Redis缓存 → 缓存未命中查数据库 → 返回字典项

字典更新流程：
管理员 → 修改字典/字典项 → 更新数据库 → 清除Redis缓存 → 下次查询从数据库读取

字典加载流程：
系统启动 → 查询所有字典 → 加载到Redis缓存 → 供业务模块使用
```

## 6. 业务规则

### 6.1 字典创建规则
1. dictCode在系统内唯一
2. dictCode命名规范：小写字母+下划线，如user_role
3. dictName必填，长度1-100字符
4. description选填，最多255字符
5. 默认状态为1（启用）
6. 系统预置字典不允许删除

### 6.2 字典项创建规则
1. 同一字典下itemCode必须唯一
2. 同一字典下itemValue必须唯一
3. itemCode命名规范：小写字母+下划线，如speaker、active
4. itemName必填，长度1-100字符
5. itemValue必填，作为TINYINT存储到业务表
6. 默认状态为1（启用）
7. sortOrder控制展示顺序

### 6.3 字典使用规则
1. 业务表使用TINYINT存储item_value
2. 展示时根据dictCode和item_value查询itemName
3. 启动的字典项才可用（status=1）
4. 禁用的字典项不会展示在选项中

### 6.4 系统预置字典规则
1. 系统启动时自动创建预置字典
2. 预置字典：user_role、user_status、submission_status、review_result、ticket_status、order_status、credential_status、common_status
3. 预置字典不允许删除

### 6.5 缓存规则
1. 字典数据变更频率低，使用Redis缓存
2. 缓存Key：dict:{dictCode}
3. 缓存过期时间：24小时
4. 字典更新时清除对应缓存
5. 系统启动时加载所有字典到缓存

## 7. 异常处理

| 异常码 | 异常信息 | 说明 |
|--------|---------|------|
| 6001 | 字典不存在 | 查询的字典不存在 |
| 6002 | 字典编码已存在 | 创建字典时编码重复 |
| 6003 | 字典格式不正确 | dictCode不符合命名规范 |
| 6004 | 系统预置字典不能删除 | 尝试删除预置字典 |
| 6005 | 字典项编码已存在 | 创建字典项时编码重复 |
| 6006 | 字典项值已存在 | 创建字典项时值重复 |
| 6007 | 字典不存在 | 创建字典项时字典不存在 |

## 8. 缓存策略

| 缓存项 | 缓存类型 | 过期时间 | 说明 |
|--------|---------|---------|------|
| dict:list | List | 24小时 | 字典列表缓存 |
| dict:{dictCode} | List | 24小时 | 单个字典的字典项列表 |
| dict:map | Map | 24小时 | 字典映射，格式Map<dictCode, Map<itemValue, itemName>> |
| dict:value:{dictCode}:{itemValue} | String | 24小时 | 根据值查询名称 |

## 9. 数据库操作

### 9.1 主要查询语句

```sql
-- 查询字典列表
SELECT * FROM dictionary
WHERE 1=1
  AND status = ?
ORDER BY sort_order ASC
LIMIT ?, ?;

-- 查询字典详情
SELECT * FROM dictionary WHERE dict_code = ?;

-- 查询字典项列表
SELECT * FROM dictionary_item
WHERE dict_id = ?
  AND status = ?
ORDER BY sort_order ASC;

-- 根据item_value查询item_name
SELECT item_name
FROM dictionary_item
WHERE dict_id = ?
  AND item_value = ?
  AND status = 1
LIMIT 1;

-- 创建字典
INSERT INTO dictionary (dict_code, dict_name, description, status, sort_order)
VALUES (?, ?, ?, 1, ?);

-- 修改字典
UPDATE dictionary
SET dict_name = ?, description = ?, sort_order = ?, updated_at = NOW()
WHERE id = ?;

-- 删除字典
DELETE FROM dictionary WHERE id = ?;

-- 创建字典项
INSERT INTO dictionary_item (dict_id, item_code, item_name, item_value, status, sort_order)
VALUES (?, ?, ?, ?, 1, ?);

-- 修改字典项
UPDATE dictionary_item
SET item_name = ?, sort_order = ?, updated_at = NOW()
WHERE id = ?;
```

### 9.2 外键维护（应用层）

```java
// 检查字典是否存在
public boolean dictExists(String dictCode) {
    return dictionaryMapper.selectCountByDictCode(dictCode) > 0;
}

// 检查字典项是否存在
public boolean dictItemExists(Long dictId, String itemCode) {
    return dictionaryItemMapper.selectCountByDictAndCode(dictId, itemCode) > 0;
}

// 根据dictCode和itemValue获取itemName
public String getItemName(String dictCode, String itemValue) {
    // 先查缓存
    String cacheKey = "dict:value:" + dictCode + ":" + itemValue;
    String cached = redisTemplate.opsForValue().get(cacheKey);
    if (cached != null) {
        return cached;
    }

    // 查数据库
    Dictionary dict = dictionaryMapper.selectByDictCode(dictCode);
    String itemName = dictionaryItemMapper.selectNameByDictAndValue(dict.getId(), itemValue);

    // 写缓存
    redisTemplate.opsForValue().set(cacheKey, itemName, 24, TimeUnit.HOURS);
    return itemName;
}
```

## 10. 缓存加载

### 10.1 系统启动加载

```java
@Component
public class DictionaryCacheLoader implements ApplicationRunner {

    @Autowired
    private DictionaryService dictionaryService;

    @Override
    public void run(ApplicationArguments args) {
        log.info("开始加载字典数据到缓存...");
        long count = dictionaryService.loadAllToCache();
        log.info("字典数据加载完成，共加载{}个字典", count);
    }
}
```

### 10.2 缓存刷新

```java
public void refreshCache(String dictCode) {
    // 清除缓存
    String cacheKey = "dict:" + dictCode;
    redisTemplate.delete(cacheKey);

    // 重新加载
    Dictionary dict = dictionaryMapper.selectByDictCode(dictCode);
    List<DictionaryItem> items = dictionaryItemMapper.selectByDictId(dict.getId());
    redisTemplate.opsForValue().set(cacheKey, items, 24, TimeUnit.HOURS);
}
```

## 11. Java枚举定义示例

### 11.1 用户角色枚举

```java
@Getter
@AllArgsConstructor
public enum UserRole {

    SPEAKER(1, "speaker", "投稿讲者"),
    REVIEWER(2, "reviewer", "审稿人"),
    ATTENDEE(3, "attendee", "参会者"),
    ADMIN(4, "admin", "管理员");

    private final Integer value;
    private final String code;
    private final String name;

    public static UserRole fromValue(Integer value) {
        for (UserRole role : values()) {
            if (role.getValue().equals(value)) {
                return role;
            }
        }
        return null;
    }

    public String getDisplayName() {
        return name;
    }
}
```
