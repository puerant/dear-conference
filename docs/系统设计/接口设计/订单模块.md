# 订单模块分析

## 1. 模块概述

订单模块负责管理用户的购票订单，包括订单创建、支付、取消、退款等完整流程。该模块是连接用户、票务和凭证的核心模块，涉及金额计算、订单状态流转、凭证生成等关键业务逻辑。

## 2. 功能模块划分

### 2.1 订单管理（所有用户）
- 创建订单
- 查看我的订单列表
- 查看订单详情
- 取消订单（仅未支付状态）
- 修改参会人信息（仅未支付状态）

### 2.2 订单支付
- 发起支付
- 支付回调处理
- 查询支付状态
- 支付超时处理

### 2.3 订单退款
- 申请退款
- 退款审核（管理员）
- 退款处理
- 退款查询

### 2.4 订单管理（管理员）
- 查询所有订单
- 查看订单详情
- 处理退款申请
- 订单统计

## 4. API实现逻辑

### 4.1 创建订单逻辑

```
1. 接收参数：ticketId, quantity, attendeeName, attendeeEmail
2. 权限校验：提取Token，获取user_id
3. 参数校验：
   - ticketId非空
   - quantity >= 1
   - attendeeName非空
   - attendeeEmail格式正确
4. 查询票务信息：
   - SELECT * FROM tickets WHERE id = ? AND status = 1
5. 检查库存：
   - available = stock - sold_count
   - available >= quantity
6. 计算总金额：
   - totalAmount = ticket.price × quantity
7. 生成订单号：
   - orderNo = "ORD" + 当前时间戳 + 6位随机数
8. 创建订单：
   - INSERT INTO orders (order_no, user_id, ticket_id, quantity, total_amount,
                        attendee_name, attendee_email, status)
   - status默认为1（未支付）
9. 返回订单信息（包含订单号）
```

### 4.2 查询订单列表逻辑

```
1. 接收参数：page, pageSize, status(可选)
2. 权限校验：提取Token，获取user_id
3. 构建查询条件：
   - WHERE user_id = ?
   - AND status = ? (status非空时)
4. 分页查询：
   - SELECT * FROM orders WHERE ... ORDER BY created_at DESC LIMIT ?, ?
5. 关联查询：
   - 关联tickets表获取票务信息
   - 关联dictionary表获取状态名称
6. 返回分页结果
```

### 4.3 查询订单详情逻辑

```
1. 接收参数：orderNo
2. 权限校验：提取Token，获取user_id
3. 查询订单：
   - SELECT o.*, t.name as ticket_name, t.price as ticket_price
     FROM orders o
     LEFT JOIN tickets t ON o.ticket_id = t.id
     WHERE o.order_no = ?
4. 验证权限：
   - 普通用户：只能查询自己的订单
   - 管理员：可以查询所有订单
5. 关联凭证表：
   - SELECT * FROM credentials WHERE order_id = ?
6. 返回订单详情
```

### 4.4 取消订单逻辑

```
1. 接收参数：orderNo
2. 权限校验：提取Token，获取user_id
3. 查询订单
4. 验证：
   - 订单存在
   - 是自己的订单
   - status为1（未支付）
5. 更新订单状态：
   - UPDATE orders SET status = 3 WHERE id = ?
6. 返回取消成功
```

### 4.5 修改参会人信息逻辑

```
1. 接收参数：orderNo, attendeeName, attendeeEmail
2. 权限校验：提取Token，获取user_id
3. 查询订单
4. 验证：
   - 订单存在
   - 是自己的订单
   - status为1（未支付）
5. 更新参会人信息：
   - UPDATE orders SET attendee_name = ?, attendee_email = ? WHERE id = ?
6. 返回更新成功
```

### 4.6 发起支付逻辑

```
1. 接收参数：orderNo, paymentMethod(微信/支付宝/银行转账)
2. 查询订单信息
3. 验证订单状态：
   - 必须为1（未支付）
4. 调用第三方支付接口：
   - 微信支付：获取支付二维码或支付链接
   - 支付宝：获取支付页面URL
5. 生成支付参数：
   - 订单号
   - 总金额
   - 商品描述
6. 返回支付参数
7. 启动支付超时定时任务（30分钟）
```

### 4.7 支付回调处理逻辑

```
1. 接收支付平台回调参数
2. 验证签名：
   - 验证回调是否来自支付平台
3. 查询订单：
   - SELECT * FROM orders WHERE order_no = ?
4. 验证订单状态：
   - 避免重复处理：status必须为1（未支付）
5. 验证支付金额：
   - 回调金额 = 订单总金额
6. 更新订单状态：
   - UPDATE orders SET status = 2, paid_at = NOW() WHERE id = ?
7. 更新票务已售数量：
   - UPDATE tickets SET sold_count = sold_count + quantity WHERE id = ?
8. 生成入会凭证：
   - 调用凭证模块生成凭证
9. 发送邮件通知：
   - 订单支付成功通知
   - 发送入会凭证到邮箱
10. 返回成功给支付平台
```

### 4.8 支付超时处理逻辑

```
1. 查询超时订单：
   - SELECT * FROM orders
     WHERE status = 1
       AND created_at < NOW() - INTERVAL 30 MINUTE
2. 更新订单状态为取消：
   - UPDATE orders SET status = 3 WHERE id IN (...)
3. 返回处理结果
```

### 4.9 申请退款逻辑

```
1. 接收参数：orderNo, reason
2. 权限校验：提取Token，获取user_id
3. 查询订单
4. 验证：
   - 订单存在
   - 是自己的订单
   - status为2（已支付）
5. 检查是否已申请退款
6. 创建退款申请记录：
   - INSERT INTO refunds (order_no, user_id, reason, status)
   - status默认为待审核
7. 返回申请成功
```

### 4.10 处理退款逻辑

```
1. 权限校验：必须为admin角色
2. 接收参数：refundId, approve(true/false), rejectReason
3. 查询退款申请
4. 审核通过：
   a. 调用支付平台退款接口
   b. 更新退款申请状态为已处理
   c. 更新订单状态为4（已退款）
   d. 更新票务已售数量：
      - UPDATE tickets SET sold_count = sold_count - quantity WHERE id = ?
   e. 取消入会凭证：
      - UPDATE credentials SET status = 3 WHERE order_id = ?
   f. 发送邮件通知用户
5. 审核拒绝：
   a. 更新退款申请状态为已拒绝
   b. 记录拒绝原因
   c. 发送邮件通知用户
6. 返回处理结果
```

### 4.11 订单统计逻辑

```
1. 权限校验：必须为admin角色
2. 查询统计信息：
   - 总订单数：SELECT count(*) FROM orders
   - 今日订单数：SELECT count(*) FROM orders WHERE DATE(created_at) = CURDATE()
   - 今日销售额：SELECT sum(total_amount) FROM orders WHERE DATE(created_at) = CURDATE() AND status = 2
   - 已支付订单数：SELECT count(*) FROM orders WHERE status = 2
   - 未支付订单数：SELECT count(*) FROM orders WHERE status = 1
   - 已取消订单数：SELECT count(*) FROM orders WHERE status = 3
   - 已退款订单数：SELECT count(*) FROM orders WHERE status = 4
3. 返回统计结果
```

## 5. 模块关联关系

### 5.1 关联的模块

| 关联模块 | 关联方式 | 关联说明 |
|---------|---------|---------|
| 用户模块 | user_id | 订单表关联下单用户 |
| 票务模块 | ticket_id | 订单表关联购买的票务 |
| 凭证模块 | order_id | 凭证表关联订单 |
| 字典模块 | status | 订单状态使用字典表维护 |

### 5.2 数据流向

```
下单流程：
用户 → 选择票务 → 创建订单接口 → orders表 → 发起支付

支付流程：
支付平台 → 支付回调 → 更新订单状态 → 生成凭证 → 发送邮件

取消流程：
用户 → 取消订单接口 → 更新订单状态 → 订单完成

退款流程：
用户 → 申请退款 → 管理员审核 → 支付平台退款 → 更新订单状态 → 取消凭证
```

## 6. 业务规则

### 6.1 订单创建规则
1. 订单号唯一，格式：ORD{timestamp}{random}
2. 参会人姓名必填
3. 参会人邮箱必填，格式需正确
4. 购票数量必须 >= 1
5. 库存不足时禁止下单
6. 只能购买status=1（上架）的票务
7. total_amount = ticket.price × quantity

### 6.2 订单状态流转
```
unpaid (未支付)
    ├─→ paid (已支付) [支付成功]
    │        └─→ refunded (已退款) [申请退款并通过]
    └─→ cancelled (已取消) [用户取消或支付超时]
```

### 6.3 支付规则
1. 只有status=1（未支付）的订单可以支付
2. 支付超时时间为30分钟
3. 支付成功后立即生成凭证
4. 支付成功后票务sold_count增加
5. 支付回调需验证签名，防止伪造

### 6.4 取消订单规则
1. 只有status=1（未支付）的订单可以取消
2. 支付中的订单不能取消
3. 已支付的订单需走退款流程
4. 用户只能取消自己的订单

### 6.5 退款规则
1. 只有status=2（已支付）的订单可以申请退款
2. 退款需管理员审核
3. 退款成功后订单状态变为refunded
4. 退款成功后票务sold_count减少
5. 退款成功后关联凭证失效

### 6.6 参会人信息规则
1. 支付前可以修改参会人信息
2. 支付后参会人信息不可修改
3. 参会人信息用于生成凭证

## 7. 异常处理

| 异常码 | 异常信息 | 说明 |
|--------|---------|------|
| 4001 | 票务不存在 | 指定的票务不存在 |
| 4002 | 票务已下架 | 无法购买已下架的票务 |
| 4003 | 库存不足 | 购买数量超过可用库存 |
| 4004 | 订单不存在 | 查询的订单不存在 |
| 4005 | 订单状态不允许取消 | 只有未支付订单可取消 |
| 4006 | 订单状态不允许支付 | 只有未支付订单可支付 |
| 4007 | 订单已支付 | 重复支付请求 |
| 4008 | 无权操作此订单 | 不是订单的下单用户 |
| 4009 | 参会人邮箱格式错误 | 邮箱格式不正确 |
| 4010 | 订单已申请退款 | 重复申请退款 |
| 4011 | 退款申请不存在 | 查询的退款申请不存在 |

## 9. 数据库操作

### 9.1 主要查询语句

```sql
-- 查询我的订单列表
SELECT o.*, t.name as ticket_name, t.price as unit_price
FROM orders o
LEFT JOIN tickets t ON o.ticket_id = t.id
WHERE o.user_id = ?
  AND o.status = ?
ORDER BY o.created_at DESC
LIMIT ?, ?;

-- 查询订单详情
SELECT o.*, t.name as ticket_name, t.price as unit_price, t.description as ticket_desc
FROM orders o
LEFT JOIN tickets t ON o.ticket_id = t.id
WHERE o.order_no = ?;

-- 查询订单的凭证
SELECT * FROM credentials WHERE order_id = ?;

-- 创建订单
INSERT INTO orders (order_no, user_id, ticket_id, quantity, total_amount,
                  attendee_name, attendee_email, status)
VALUES (?, ?, ?, ?, ?, ?, 1);

-- 更新订单状态（支付成功）
UPDATE orders
SET status = 2, paid_at = NOW()
WHERE id = ?;

-- 更新订单状态（取消）
UPDATE orders
SET status = 3
WHERE id = ?;

-- 更新订单状态（退款）
UPDATE orders
SET status = 4
WHERE id = ?;

-- 更新参会人信息
UPDATE orders
SET attendee_name = ?, attendee_email = ?
WHERE id = ?;

-- 查询超时订单
SELECT * FROM orders
WHERE status = 1
  AND created_at < DATE_SUB(NOW(), INTERVAL 30 MINUTE);

-- 订单统计
SELECT
    count(*) as total_count,
    sum(CASE WHEN status = 1 THEN 1 ELSE 0 END) as unpaid_count,
    sum(CASE WHEN status = 2 THEN 1 ELSE 0 END) as paid_count,
    sum(CASE WHEN status = 3 THEN 1 ELSE 0 END) as cancelled_count,
    sum(CASE WHEN status = 4 THEN 1 ELSE 0 END) as refunded_count,
    sum(CASE WHEN status = 2 THEN total_amount ELSE 0 END) as total_sales
FROM orders;
```

public boolean orderExists(String orderNo) {

## 10. 订单号生成

### 10.1 生成规则

格式：`ORD{timestamp}{random}`

示例：`ORD20260225143000123456`

- ORD：订单前缀
- 20260225143000：时间戳14位（年月日时分秒）
- 123456：6位随机数

### 10.2 生成代码

```java
public String generateOrderNo() {
    long timestamp = System.currentTimeMillis();
    String timestampStr = String.valueOf(timestamp);
    String random = RandomStringUtils.randomNumeric(6);
    return "ORD" + timestampStr.substring(timestampStr.length() - 14) + random;
}
```
